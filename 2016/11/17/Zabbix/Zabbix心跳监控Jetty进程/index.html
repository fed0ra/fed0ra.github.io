Zabbix心跳监控Jetty进程


Welcome to Fedor’s blog! If you find any problems, you can tell me in any way.

1. 克隆代码

# git clone https://github.com/fed0ra/template4zabbix.git

2. 拷贝配置文件

# cp template4zabbix/zabbix-agent/conf/ping_jetty.conf /etc/zabbix/zabbix_agentd.d/

3. 拷贝脚本

# 创建存放脚本目录
# mkdir -p /etc/zabbix/zabbix_shells
# 拷贝脚本文件
# cp template4zabbix/zabbix-agent/scripts/ping_jetty.sh /etc/zabbix/zabbix_shells/

4. 编辑Zabbix Agent配置文件

# 修改内容如下
# vim /etc/zabbix/zabbix_agentd.conf
Include=/etc/zabbix/zabbix_agentd.d/*.conf
UnsafeUserParameters=1

# 重启客户端agentd
# /etc/init.d/zabbix-agent restart

5. 客户端测试脚本可用性

# cd /etc/zabbix/zabbix_shells/
# sh ping_jetty.sh jetty_ping 8080 testproject
1

jetty_status为key，8080是所监控的jetty端口，testproject为工程名（没有工程名可不写）

默认进程存在输出1，进程不存在输出0】

6. 服务端获取参数

# zabbix_get -s 10.0.0.10 -p 10050 -k jetty_status[jetty_ping,8080,testproject]
1

-s代表被监控服务端的Zabbix agent IP地址。
-p代表被监控服务端的Zabbix agent端口。

7. 登录Zabbix web页面导入模板

模板在template4zabbix/zabbix-server/路径下。

Jetty模板：template4zabbix/zabbix-server/templates/Template Java Jetty.xml

需监控多个jetty进程，通过，配置–>模板–>Template Java Jetty，可自定义添加应用集、监控项和触发器。
