    Zabbix监控Node.js


    Welcome to Fedor’s blog! If you find any problems, you can tell me in any way.

    通过pm2-zabbix工具可以实现Zabbix对Node.js的监控报警。
    Github地址https://github.com/greatcare/pm2-zabbix

    • 自动发现通过PM2管理的Node.js进程
    • 报告Nodes.js进程状态、CPU占用率、内存占用以及进程是否重启
    • 监控PM2守护进程自身状态、资源占用和PID改变
    • 已提供易于安装的Zabbix监控项模板

    一、Node.js服务器安装pm2-zabbix

    # npm install -g pm2-zabbix

    二、测试自动发现功能、连通性

    1. 测试自动发现功能

    使用pm2 ls查看启动的nodejs进程，然后使用pm2-zabbix --discover命令，输出如下类似字典表明自动发现功能正常：

    2. 测试连通性

    异步后台监控协议使用zabbix_sender发送数据到服务器。

    默认配置参数来自于/etc/zabbix/zabbix_agentd.conf，包括服务器地址和身份验证凭据等。

    开启监控模式，使用如下命令：

    [root@test ~]# pm2-zabbix --monitor

    将启动一个线程，连接到当前用户的PM2实例，并在后台发送更新，pm2-zabbix --help查看帮助。

    三、克隆代码

    [root@test ~]# git clone https://github.com/greatcare/pm2-zabbix.git

    四、添加服务

    pm2-zabbix仅仅是一个能被pm2启动的Node.js脚本，这样的做法是不推荐的，推荐将pm2-zabbix添加到服务。

    # RHEL/CentOS 6
[root@test ~]# cp pm2-zabbix/install/init/sysv/pm2-zabbix /etc/init.d/
# 修改启动文件pm2-zabbix内容，改为`DAEMON_USER=root`
[root@test ~]# vim /etc/init.d/pm2-zabbix

# RHEL/CentOS 7
[root@test ~]# cp pm2-zabbix/install/init/systemd/pm2-zabbix.service /usr/lib/systemd/system
# 修改启动文件pm2-zabbix.service内容，改为`User=root`

    五、配置Zabbix Agent和Zabbix Server

    1. 从源代码中拷贝zabbix-agent的配置文件

    [root@test ~]# cp pm2-zabbix/install/zabbix-agent/pm2-zabbix.conf /etc/zabbix/zabbix_agentd.conf.d/
[root@test ~]# cat /usr/local/zabbix-3.0.4/etc/zabbix_agentd.conf.d/pm2-zabbix.conf
# Put this file in /etc/zabbix/zabbix_agentd.d/ and remember to change the sudo -u <user> to use your user name
UserParameter=pm2.processes,sudo -u root pm2-zabbix --discover

# Note that it is advised you test whether sudo works from the zabbix user first, including *without a TTY*.
# If you get a "Value should be a JSON object" error during key discovery, please check your sudoers -
# refer to the example sudoers file for an option that disables the tty requirement.
[root@test ~]#

    2. Zabbix Server导入模板，模板文件路径

    [root@test ~]# ll pm2-zabbix/install/zabbix-server/pm2-zabbix.template.xml

    六、创建sudo用户zabbi

    [root@test www]# cp pm2-zabbix/install/zabbix-agent/pm2-zabbix-sudoers /etc/sudoers.d/

    七、测试获取数据

    在Zabbix-agent端使用zabbix-get（需安装）命令测试是否能获取数据。


    -s根参数跟Zabbix agentIP地址，-p端口。

    如果输出信息，没有报错即可。
