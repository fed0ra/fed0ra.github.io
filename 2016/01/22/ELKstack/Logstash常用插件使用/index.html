    Logstash常用插件使用


    Welcome to Fedor’s blog! If you find any problems, you can tell me in any way.

    学习一个插件的时候，先将结果输入到标准输出，启动logstash后插件是否生效（如下），不要急于求成一次性写到elasticsearch中，如果出现问题日志收集不成功时不好排错，测试没问提后再写到elasticsearch中。

    input {
    file {
        path => "/var/log/messages"
        type => "system-massages"
        start_position => "beginning"
    }

}

output {
    stdout {
        codec => "rubydebug"
    }

}

    logstash常用插件使用:

    1.使用logstash常用几种插件收集系统日志、syslog日志、java日志、nginx访问日志。

    ####示例：编写配置文件cat /etc/logstash.conf

    #输入
input {
    #使用文件方式收集system log
    file {
        path => "/var/log/messages"
        type => "system-massages"
        start_position => "beginning"
    }
    #codec multiline插件收集java错误日志
    file {
        path => "/path/jetty/logs/*"
        type => "java-error"
        #start_position => "beginning"
        codec => multiline {
            pattern => "^[0-1][0-9]:[0-5][0-9]|2[0-3]:[0-5][0-9]|^\["
            negate => true
            what => previous
        }
    }
    #codec json插件手机json日志
    file {
        path => "/path/to/your/nginxlog/access-json.log"
        codec => "json"
        type => "nginx_log"
        start_position => "beginning"
    }
    #使用input syslog插件收集系统日志，此插件会监控tcp和udp的514端口，使用此插件需要设置rsyslog，让rsyslog把日志发送的514端口上，设置见下文
    syslog {
            type => "system-syslog"
            host => "192.168.2.42"
            port => "514"
    }

}

#输出
output {
    #根据key类型判断，写入到elasticsearch的哪一个key中
    if [type == "system-massages"] {
        elasticsearch {
            hosts => "192.168.2.42"
            index => "system-massages-%{+YYYY.MM.dd}"
        }
    }

    if [type] == "java-error"  {
        elasticsearch {
            hosts => "192.168.2.42"
            index => "java-error-%{+YYYY.MM.dd}"
        }
    }

    if [type] == "nginx_log"  {
        elasticsearch {
            hosts => "192.168.2.42"
            index => "nginx_log-%{+YYYY.MM.dd}"
        }
    }

    if [type] == "system-syslog"  {
        elasticsearch {
            hosts => "192.168.2.42"
            index => "system-syslog-%{+YYYY.MM.dd}"
        }
    }      
}

    设置rsyslog方法：编辑配置文件vim /etc/rsyslog.conf，去掉下面指定行的注释，修改remote-host的IP地址指定某一台机器去收集

    #*.* @@remote-host:514
*.* @@192.168.2.42:514

    2.tcp日志和tcp插件

    场景：把某些日志追加到es的索引中。
    方法：使用logstash监听一个tcp端口，写一个脚本，通过nc命令把文件内容发送到后端es中的某个索引即可。

    input {
    tcp {
        host => "192.168.2.42"
        port => "6666"
    }
}

output {
    stdout {
        codec => "rubydebug"
    }
}

    启动logstash后，netstat -ntlp查看6666端口已经起来了。

    使用nc命令发送文件

    # nc 192.168.2.42 6666 < /etc/resolv.conf
# echo "new line" | nc 192.168.2.42 6666
#伪设备、伪终端
# echo "two line" > /dev/tcp/192.168.2.42/6666

    3.grok插件

    官网例子：

    [root@test ~]# cat grok.conf 
input {
    stdin {}
}
#filter
filter {
    #grok插件，通过正则匹配，在HTTP日志中取出有用的字段，以特定格式输出。
    #比如apache日志不能像nginx一样输出json格式，可以使用grok插件自定义。
    grok {
        match => { "message" => "%{IP:client} %{WORD:method} %{URIPATHPARAM:request} %{NUMBER:bytes} %{NUMBER:duration}" }
    }
}

output {
    stdout {
        codec => "rubydebug"
    }
}
[root@test ~]#

    启动logstash，输入一段虚构的http请求日志：55.3.244.1 GET /index.html 15824 0.043，会以以下格式输出。

    实例：收集MySQL slowlog日志

    查看/tmp/mysqlslow.log

    [root@test data]# tail -f /tmp/mysqlslow.log
# Time: 130801  2:36:25
# User@Host: root[root] @ localhost []
# Query_time: 2.274219  Lock_time: 0.000322 Rows_sent: 0  Rows_examined: 5440
use test;
SET timestamp=1375295785;
insert into t select * from t;

    分析slow.log格式，从Time开始算是一个事件，所以要用到多行插件（multiline）把多行处理成单个事件。
    编辑配置文件：

    [root@test ~]# cat mysql-slow.conf 
input {
    file {
        path => "/tmp/mysqlslow.log"
        type => "mysql-slowlog"
        codec => multiline {
            pattern => "^# User@Host:"
            negate => true
            what => previous
        }
    }
}

filter {
    grok {
        match => [ "message" , "(?m)^# User@Host: %{USER:user}\[[^\]]+\] @ (?:(?<clienthost>\S*) )?\[(?:%{IP:clientip})?\]\s+Id: %{NUMBER:row_id:int}\s*# Query_time: %{NUMBER:query_time:float}\s+Lock_time: %{NUMBER:lock_time:float}\s+Rows_sent: %{NUMBER:rows_sent:int}\s+Rows_examined: %{NUMBER:rows_examined:int}\s*(?:use %{DATA:database};\s*)?SET timestamp=%{NUMBER:timestamp};\s*(?<query>(?<action>\w+)\s+.*)\n#\s*" ]
    }
    date {
        match => [ "timestamp", "UNIX" ]
        remove_field => [ "timestamp" ]
    }
}

output {
    stdout {
        codec => "rubydebug"
    }
}
[root@test ~]# 
[root@test ~]# /opt/logstash/bin/logstash -f mysql-slow.conf --configtest
Configuration OK
[root@test ~]# /opt/logstash/bin/logstash -f mysql-slow.conf

    启动logstash，向/tmp/mysqlslow.log写入日志，会输出到屏幕上。
    grok缺点，当日志量特别大的时候很占用CPU。
