一、Linux上安装部署redis

1.下载redis

# wget http://download.redis.io/releases/redis-3.0.7.tar.gz

2.解压编译安装

# tar xzf redis-3.0.7.tar.gz
# cd redis-3.0.7/src
# make
# make install

此时可以看到src文件夹下多出现了一些绿色的文件，这些文件就是我们以后需要用到的命令文件。

3.为了便于管理，所有源代码安装的软件都安装在/usr/local/redis下

创建bin,etc,var三个文件夹（bin用于存放命令，etc拥有存放配置文件，var用于存放日志文件）
将redis-3.0.7源码文件夹下的redis.conf复制到/usr/local/redis/etc/下
将src目录下的7个命令文件（绿色的），移动到/usr/local/redis/bin/下

# mkdir -p /usr/local/redis/{bin,etc,var}
# /bin/cp ./{redis-benchmark,redis-check-aof,redis-check-dump,redis-cli,redis-sentinel,redis-server} /usr/local/redis/bin/
# cd ../
# cp ./redis.conf /usr/local/redis/etc/
# ln -s /usr/local/redis/bin/* /usr/local/bin/
# cd ..
# rm -rf redis-3.0.7

这样启动没有用到etc下的配置文件，并且服务是在前台运行。

通过配置文件后台启动redis
ctrl+C停止服务。
修改redis配置文件，如果需要在后台运行，修改daemonize参数改为yes，修改命令如下：

sed -i 's@pidfile.*@pidfile /var/run/redis.pid@' /usr/local/redis/etc/redis.conf
sed -i "s@logfile.*@logfile $redis_install_dir/var/redis.log@" /usr/local/redis/etc/redis.conf
sed -i "s@^dir.*@dir $redis_install_dir/var@" /usr/local/redis/etc/redis.conf
sed -i 's@daemonize no@daemonize yes@' /usr/local/redis/etc/redis.conf

启动redis

# redis-server /usr/local/redis/etc/redis.conf

查看是否启动成功：

# netstat -lntup | grep 6379

5.客户端登录

执行命令

# redis-cli

6.关闭redis服务

可以使用pkill命令

# pkill redis-server
# netstat -lntup | grep 6379
# pstree -p | grep redis

关闭之后，发现6379就不再被占用了，redis的进程也都没有了。
也可以使用客户端命令redis-cli

# redis-cli shutdown
# netstat -lntup | grep 6379
# pstree -p | grep redis

二、redis用户验证设置

1.设置验证密码方法一：

# vim /usr/local/redis/etc/redis.conf

修改配置文件redis.conf，设置如下

# Warning: since Redis is pretty fast an outside user can try up to
# 150k passwords per second against a good box. This means that you should
# use a very strong password otherwise it will be very easy to break.
#
requirepass 123456

2.设置验证密码方法二：

通过config命令可以设置密码以及获取当前用户密码

127.0.0.1:6379> config set requirepass 123456
OK
127.0.0.1:6379> config get requirepass 
1) "requirepass"
2) "123456"
127.0.0.1:6379>

设置验证密码之后，redis服务关闭需要密码验证通过之后才能关闭，命令如下：

# redis-cli –a 123456 shutdown

登录使用-a参数，命令如下：

# redis-cli -a 123456

来自：http://blog.csdn.net/vtopqx/article/details/46833099

三、redis作为缓存服务器的配置

用redis作为缓存服务器配置时候需要注意几点：

  • 作为缓存服务器，如果不加以限制内存的话，很可能出现将服务器内存都耗光的情况，可以编辑redis配置文件修改maxmemory参数的大小限制内存：

    # In short... if you have slaves attached it is suggested that you set a lower
# limit for maxmemory so that there is some free RAM on the system for slave
# output buffers (but this is not needed if the policy is 'noeviction').
# 限定最多使用1.5GB内存
maxmemory 1536mb

  • 如果内存到达了指定的上限，还要往redis里面添加更多的缓存内容，需要设置清理内容的策略：

    # MAXMEMORY POLICY: how Redis will select what to remove when maxmemory
# is reached. You can select among five behaviors:
#
# volatile-lru -> remove the key with an expire set using an LRU algorithm
# allkeys-lru -> remove any key according to the LRU algorithm
# volatile-random -> remove a random key with an expire set
# allkeys-random -> remove a random key, any key
# volatile-ttl -> remove the key with the nearest expire time (minor TTL)
# noeviction -> don't expire at all, just return an error on write operations
#
# Note: with any of the above policies, Redis will return an error on write
#       operations, when there are no suitable keys for eviction.
#
#       At the date of writing these commands are: set setnx setex append
#       incr decr rpush lpush rpushx lpushx linsert lset rpoplpush sadd
#       sinter sinterstore sunion sunionstore sdiff sdiffstore zadd zincrby
#       zunionstore zinterstore hset hsetnx hmset hincrby incrby decrby
#       getset mset msetnx exec sort
#
# The default is:
#
maxmemory-policy allkeys-lru

    清理策略有多种，上边贴出的配置文件里有介绍
    redis的官方文档也有说明： http://redis.io/topics/lru-cache

四、监控Redis状态

redis提供了INFO命令，能够监控redis服务器的状态
telnet到对应服务器的端口，执行命令：

[root@CentOS-Server01 src]# telnet localhost 6379
Trying ::1...
Connected to localhost.
Escape character is '^]'.
info
-NOAUTH Authentication required.
auth 123456
+OK
info
$1953
# Server
redis_version:3.0.7
redis_git_sha1:00000000
redis_git_dirty:0

如果需要验证，输入auth password即可

也可以通过以下命令获取memory usage信息和hits信息

# redis-cli -a 123456 info memory
# redis-cli -a 123456 info stats

在输出的信息里面有这几项和缓存的状态比较有关系：

keyspace_hits:14414110
keyspace_misses:3228654
used_memory:433264648
expired_keys:1333536
evicted_keys:1547380

通过hits和miss，可以得到缓存的命中率：14414110 / (14414110 + 3228654)=81% ，一个缓存失效机制，和过期时间设计良好的系统，命中率可以做到95%以上,对于整体性能提升是很大的。
used_memory,expired_keys,evicted_keys这3个信息的具体含义，redis的官方说明：http://redis.io/commands/info

有个ruby gem叫redis-stat，它利用INFO命令展现出更直观的信息报表，项目地址：https://github.com/junegunn/redis-stat

来自：https://ruby-china.org/topics/22761#reply7

五、redis的配置文件内容

redis的配置文件内容：

daemonize 如果需要在后台运行，把该项改为yes
pidfile 配置多个pid的地址 默认在/var/run/redis.pid
bind 绑定ip，设置后只接受来自该ip的请求
port 监听端口，默认是6379
loglevel 分为4个等级：debug verbose notice warning
logfile 用于配置log文件地址
databases 设置数据库个数，默认使用的数据库为0
save 设置redis进行数据库镜像的频率。
rdbcompression 在进行镜像备份时，是否进行压缩
dbfilename 镜像备份文件的文件名
Dir 数据库镜像备份的文件放置路径
Slaveof 设置数据库为其他数据库的从数据库
Masterauth 主数据库连接需要的密码验证
Requriepass 设置 登陆时需要使用密码
Maxclients 限制同时使用的客户数量
Maxmemory 设置redis能够使用的最大内存
Appendonly 开启append only模式
Appendfsync 设置对appendonly.aof文件同步的频率（对数据进行备份的第二种方式）
vm-enabled 是否开启虚拟内存支持   （vm开头的参数都是配置虚拟内存的）
vm-swap-file 设置虚拟内存的交换文件路径
vm-max-memory 设置redis使用的最大物理内存大小
vm-page-size 设置虚拟内存的页大小
vm-pages 设置交换文件的总的page数量
vm-max-threads 设置VM IO同时使用的线程数量
Glueoutputbuf 把小的输出缓存存放在一起
hash-max-zipmap-entries 设置hash的临界值
Activerehashing 重新hash

来自：http://www.open-open.com/lib/view/open1426468117367.html
