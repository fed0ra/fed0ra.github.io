    一、salt简介

    salt介绍：

    • salt是一个基础平台管理工具
    • salt是一个分布式远程执行系统，用来在远程节点上执行命令和查询数据
    • 是一个配置管理系统，能够维护预定义状态的远程节点

    salt的核心功能：

    • 使用命令发送到远程系统，并行非串行
    • 使用安全加密协议
    • 使用最小最快的网络载荷
    • 提供简单的编程接口

    salt优点

    • salt用python编写，轻量级
    • 通讯层采用ZeroMQ实现
    • 开源，通过python写模块二次开发

    Salt-master启动时会启动两个端口

    • 4505：是salt的消息发布专用端口
    • 4506：是客户端与服务端通信的端口*

    二、salt安装部署

    1.安装epel

    CentOS 6 64位

    # rpm -Uvh http://mirrors.yun-idc.com/epel/6Server/x86_64/epel-release-6-8.noarch.rpm

    CentOS 5 64位

    # rpm -Uvh http://mirrors.yun-idc.com/epel/5Server/x86_64/epel-release-5-4.noarch.rpm

    2.安装Salt

    只在server端安装master，执行命令

    # yum -y install salt-master

    然后在客户端安装minion，执行命令

    #yum -y install salt-minion

    3.配置文件及解析

    master端配置文件暂时不需要修改
    minion端master和id两个参数，暂不修改

    # cat /etc/salt/minion
master: master的ip或者域名，默认salt
id: minion自己所处的角色，默认不写是自己的主机名

    修改所有minion，使解析salt短域名解析到salt-master的ip（也就是192.168.174.10）

    # vim /etc/hosts
192.168.174.10 salt    \\salt-master的ip是192.168.174.10

    4.启动salt-master和salt-minion

    启动salt-master，查看4505和4506端口是否存在

    # /etc/init.d/salt-master start    #服务端端启动master
# netstat -lntup
Active Internet connections (only servers)
Proto Recv-Q Send-Q Local Address               Foreign Address             State       PID/Program name   
tcp        0      0 0.0.0.0:22                  0.0.0.0:*                   LISTEN      1169/sshd           
tcp        0      0 0.0.0.0:4505                0.0.0.0:*                   LISTEN      1387/python2.6              
tcp        0      0 0.0.0.0:4506                0.0.0.0:*                   LISTEN      1407/python2.6      
tcp        0      0 :::22                       :::*                        LISTEN      1169/sshd

    启动salt-minion

    # /etc/init.d/salt-minion start    #客户端启动minion

    启动的时候在minion端的配置文件目录里生成minion_id，它会缓存minion自己的id

    注：如果某一批机器已在salt中存在，而这些minion因为业务或其他关系等，需要改变主机名，如下操作：
    第一，先使用hostname newname临时生效。
    第二，修改vim /etc/sysconfig/network使重启也生效。
    第三，清空客户端minion_id文件的内容和pki目录。
    第四，在salt-master使用salt-key -d old-key -y命令把旧的key给剔除。
    第五，重启salt-minion服务，此时salt-master执行salt-key -L就会出现新修改的主机名，再在master上认证。

    [root@CentOS-Server02 ~]# cat /etc/salt/minion_id 
CentOS-Server02.example.com

    另外字pki目录下会生成公钥和私钥，在master上认证后会生成minion_master.pub

    [root@CentOS-Server02 ~]# ll /etc/salt/pki/minion/
total 12
-rw-r--r-- 1 root root  451 Feb 23 23:35 minion_master.pub
-r-------- 1 root root 1675 Feb 23 23:29 minion.pem
-rw-r--r-- 1 root root  451 Feb 23 23:29 minion.pub

    三、salt命令

    1.salt-key [-L,-a,-A,-r,-R,-d,-D,-y]

    -L列出所有的key

    [root@salt salt]# salt-key -L
Accepted Keys:
Denied Keys:
Unaccepted Keys:
CentOS-Server02.example.com
CentOS-Server03.example.com
Rejected Keys:

    刚刚配置好的出两台minion，主机名已经列出来了，状态是Unaccepted Keys，表示未被接受的key

    -a允许某台机器加进来，-A是允许所有的，-y代表yes
    现在添加所有key，命令如下：

    [root@salt salt]# salt-key -A -y
The following keys are going to be accepted:
Unaccepted Keys:
CentOS-Server02.example.com
CentOS-Server03.example.com
Key for minion CentOS-Server02.example.com accepted.
Key for minion CentOS-Server03.example.com accepted.

    也可以使用-a单独添加

    # salt-key -a CentOS-Server01 -y

    查看所有的key就添加上了，状态变为Accepted Keys

    [root@salt salt]# salt-key -L
Accepted Keys:
CentOS-Server02.example.com
CentOS-Server03.example.com
Denied Keys:
Unaccepted Keys:
Rejected Keys:

    2.对minion端发起命令

    测试minion是否通，命令如下

    [root@salt salt]# salt '*' test.ping
CentOS-Server03.example.com:
    True
CentOS-Server02.example.com:
    True

    *是匹配所有minion，test.ping是执行test模块里的ping方法，会返回所有ping minion的结果，为True或者为False。
    流程是：首先master会根据minion的id进行正则匹配，匹配所有纳入管理的机器，告诉minion上执行test.ping的动作，之后，minion端接受到指令，会ping master，并给返回结果。

    四、变更salt-master

    场景：salt-minion不断增多，导致salt-master压力不断增大，需更换性能好的服务器作为salt-master。

    原理：master和minion启动服务的时候都会生成自己的key，认证后master会把自己的key给minion端，所以minion端保存两个key，一个是自己的，一个是master。

    需要注意的是：如果key稍微出现异常（master key和minion保存的master key有冲突）的话，minion端会把自己的服务停掉，如果minion过多的情况下（成百上千太），minion都停掉的话，迁移会造成很大的麻烦，所以需要在保证minion不停的情况下，将salt-master更换掉是必要的。还有就是指minion向的ip一定要是master。

    1.在新准备的服务器上安装salt-master

    # rpm -Uvh http://mirrors.yun-idc.com/epel/6Server/x86_64/epel-release-6-8.noarch.rpm
# yum install -y salt-master

    2.打包salt-master的pik，传到新的master上

    # cd /etc/salt/
# tar zcvf pki.tar.gz pki/
# scp -r pki.tar.gz root@192.168.174.14:/etc/salt/
\\192.168.174.14是新的master的ip地址

    3.去新的salt-master上解压，并启动master

    # tar zxvf pki.tar.gz
# /etc/init.d/salt-master start

    master的启动不会做任何操作,只判断自己的key有没有，没有就生成。

    4.把minion端配置的master的ip改变

    需要在master上批量改变minion端配置文件里面的指定的master的ip。
    salt提供cmd.run命令，在linux上执行命令，例如查看linux的发行版本，如下命令：

    # salt '*' cmd.run 'cat /etc/redhat-release'

    批量修改所有minion的解析为新的salt-master的地址：

    # salt '*' cmd.run "cat /etc/hosts"
# salt '*' cmd.run "cat /etc/hosts | grep salt | sed -i 's/192.168.174.10/192.168.174.14/' /etc/hosts"
# salt '*' cmd.run "cat /etc/hosts"

    重启所有minion服务，可以是salt提供的启动服务的模块

    # salt '*' service.restart salt-minion

    验证salt-master是否可以进行操作

    # salt '*' cmd.run 'cat /etc/hosts | grep salt'

    登录到新的master，验证新的salt-master是否可以进行操作

    [root@CentOS-Server04 salt]# salt '*' cmd.run 'cat /etc/hosts | grep salt'
CentOS-Server03.example.com:
    192.168.174.14 salt
CentOS-Server02.example.com:
    192.168.174.14 salt

    把salt-master自己生成的key、已认证的、未认证的key，都打包拿到新的salt-master上来，之后批量重启服务，自动全部获取。

    五、event和returnner

    1.returnner

    master发送命令给minion后，会将执行结果返回给master，而saltstack returnner接口可以将结果发送给任意系统

    内置returnner模块：

    mysql:将结果返回给mysql
mongo_future_return:返回给mongodb
mongo_return:同上
redis_return:返回给redis
carbon_return:carbon接收器
cassandra_return:返回给Cassandra columnfamily
local:返回给本地的测试returnner接口，即打印当前结果
sentry_return:返回给sentry
smtp_return:通过email返回salt数据
syslog_return:返回结果给操作系统的syslog facility

    把结果存在日志中
    执行命令时候加--return参数

    # salt '*' test.ping --return syslog

    结果会存到每台minion各自的系统日志中，查看minion的系统日志，会发现：

    # tail /var/log/messages
Feb 24 04:32:36 CentOS-Server02 python2.6: {"fun_args": [], "jid": "20160224043235810137", "return": true, "retcode": 0, "success": true, "fun": "test.ping", "id": "CentOS-Server02.example.com"}

    2.event

    event是本地的ZeroMQ pub Interace，一个开放系统，发送信息通知salt或其他操作系统。

    实现在master上把数据进入数据库。minion把信息发给master的event，event通知salt，salt把结果写入终端、数据库等。操作步骤如下：

    准备MySQL数据库，创建数据库，授权：

    准备数据库，可以再master，也可以在其他服务器上：

    # yum install mysql mysql-server -y
# /etc/init.d/mysqld start
# /usr/bin/mysqladmin -u root password '123456'
# mysql -uroot -p

    创建数据库：

    yum in
CREATE DATABASE `salt`
  DEFAULT CHARACTER SET utf8
  DEFAULT COLLATE utf8_general_ci;

USE `salt`;

--
-- Table structure for table `jids`
--

DROP TABLE IF EXISTS `jids`;
CREATE TABLE `jids` (
  `jid` varchar(255) NOT NULL,
  `load` mediumtext NOT NULL,
  UNIQUE KEY `jid` (`jid`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

--
-- Table structure for table `salt_returns`
--

DROP TABLE IF EXISTS `salt_returns`;
CREATE TABLE `salt_returns` (
  `fun` varchar(50) NOT NULL,
  `jid` varchar(255) NOT NULL,
  `return` mediumtext NOT NULL,
  `id` varchar(255) NOT NULL,
  `success` varchar(10) NOT NULL,
  `full_ret` mediumtext NOT NULL,
  `alter_time` TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  KEY `id` (`id`),
  KEY `jid` (`jid`),
  KEY `fun` (`fun`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

    授权（本机不用授权）：

    mysql> grant all on salt.* to salt@'192.168.174.%' identified by "salt123";
mysql> flush privileges;
mysql> select user,host,password from mysql.user;

    在master端主备监听event的python脚本

    将监听到的数据写入数据库，编写监听的python脚本：

    [root@CentOS-Server01 ~]# cat salt_event_to_mysql.py 
#!/bin/env python
#coding=utf8

#Import python libs
import json
#Import salt modules
import salt.config
import salt.utils.event
#Import third party libs
import MySQLdb

__opts__ = salt.config.client_config('/etc/salt/master')

#Create MySQL connect
conn = MySQLdb.connect(host=__opts__['mysql.host'],user=__opts__['mysql.user'],passwd=__opts__['mysql.pass'],db=__opts__['mysql.db'],port=__opts__['mysql.port'])
cursor = conn.cursor()

#Listen Salt Master Event System
event = salt.utils.event.MasterEvent(__opts__['sock_dir'])
for eachevent in event.iter_events(full=True):
    ret = eachevent['data']
    if "salt/job/" in eachevent['tag']:
        #Return Event
        if ret.has_key('id') and ret.has_key('return'):
            #Igonre saltutil.find_job event
            if ret['fun'] == "saltutil.find_job":
                continue

            sql = '''INSERT INTO `salt_returns`
                (`fun`, `jid`, `return`, `id`, `success`, `full_ret`)
                VALUES (%s, %s, %s, %s, %s, %s)'''
            cursor.execute(sql, (ret['fun'], ret['jid'],
                                json.dumps(ret['return']), ret['id'],
                                ret['success'], json.dumps(ret)))
            cursor.execute("COMMIT")
    #Other Event
    else:
        pass

    修改master配置文件，添加到最后、配置如下（根据实际情况修改）：

    mysql.host: '192.168.174.14'
mysql.user: 'salt'
mysql.pass: 'salt123'
mysql.db: 'salt'
mysql.port: 3306

    在master上安装mysql和python的MySQL模块：

    # yum install mysql -y
# yum install MySQL-python -y

    在master上执行监听脚本：

    # python salt_event_to_mysql.py &
