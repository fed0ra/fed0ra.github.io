    一、Eclipse+PyDev插件搭建Python开发环境

    编辑器: Eclipse + pydev插件

    1. Eclipse是写JAVA的IDE， 这样就可以通用了,学习代价小。 学会了Eclipse, 以后写Python或者JAVA 都可以。
    2. Eclipse, 功能强大。
    3. Eclipse跨平台， 可以在Mac上和Windows运行

    1.准备工作

    • 下载Python
      下载地址：http://www.python.org/
      Python有Python2和Python3两个版本，语法有些区别。保险起见,我安装Python 2.7.6
    • 下载JAVA JDK
      下载地址：http://www.oracle.com/technetwork/java/javase/downloads/index.html
      Eclipse 需要这个安装好JAVA JDK后才能运行
    • 下载Eclipse
      下载地址：http://www.eclipse.org/downloads/
      下载完后，解压就可以直接使用，Eclipse不需要安装。
    • pydev插件介绍
      pydev插件的官方网站: http://www.pydev.org/（不需要下载）

    2.安装JDK6

    JDK的安装很容易的，一般一直点下一步即可。多数教程都说，安装完JDK需要设置环境变量，但如今的IDE已经不再通过环境变量来查找JDK的路径了，所以这个不设也罢。

    3.安装Eclipse和Python

    Eclipse是绿色软件，找个地方解压一下就OK了 。Python的安装就更不用说了，一般点下一步即可。

    4.给Eclipse安装PyDev插件

    启用Eclipse，如果前面的步骤都正确无误的话，那么Eclipse就该能够正常启动了。 第一次启动会让你选择一个工作空间，按缺省设置，勾选一下不再提醒，就可以了。
    如果启动Eclipse打开出现“no java virtual machine was found...."，或Eclipse无法打开“Failed to load the JNI shared library”看文章第二部分

    方法一：（PyDev的更新地址不好确定，一般使用方法二）
    　　启动Eclipse后，在Help菜单中，选择Install New Software···， 选择Add按钮，Name：PyDev（这个随便起），Location：http://pydev.org/updates（PyDev的更新地址），OK
    　　选择PyDev下的PyDev for Eclipse，别的都不要选，否则依赖检查那关过不去。点下一步，下一步安装。中间会问你是否信任Aptana、Pydev什么的，信任就可以了。
    　　最后重启Eclipse

    方法二：
    在Window菜单中选Help -> Eclipse Marketplace -> Search - PyDev - Installed，等待。

    5.配置PyDev插件

    在Window菜单中选Preference

    • PyDev->Interpreter - Python，New一个Python解释器，填上解释器名字和路径，路径选相应的python.exe。
      以下几步Python 3.x必选，Python2随意。比较习惯Unicode，所以一次性在这儿设定了。
    • General->Editors->Text Editors->Spelling，Encoding改成Other:UTF-8
    • General->Workspace，Text file encoding改成Other:UTF-8
    • 找到安装目录下的\plugins\org.python.pydev.debug_x.x.x.yyyymmddhh\pysrc\pydevd.py
      920行的encoding = Non改成encoding = "UTF-8"，保存。

    至此，PyDev的配置就完成了。

    6.测试

    File->New->Project，选PyDev下的PyDev Project，Grammer和Interpreter选相应的版本，Finish。
    在PyDev Package Explorer的项目上右键，New->PyDev Module，随便写个名字，Finish。
    然后随便写几行代码，Run
    在弹出的对话框中选择Python Run
    OK

    7.注释

    注1：这里说明一下，JDK和Eclispse好像得同为32位或者64位的，否则有可能遇到问题。我第一次尝试是用的64位JDK+32位Eclipse的组合，就碰到了和这里http://superuser.com/questions/186001/eclipse-failed-to-load-the-jni-shared-library一样的问题，于是换了32位JDK。

    注2：我想做GAE的开发，所以选了版本2.6.6，一般情况下选最新版即可。

    注3：PyDev Mylyn Integration (optional)：这个依赖于PyDev for Eclipse，你得装完后者就可以装了。 PyDev Django Templates Editor：这个需要Aptana Studio 3。

    二、Eclipse报错问题

    1.Eclipse打开出现no java virtual machine was found
    这个错误一般是环境变量设置错误导致：
    在点击计算机开始–>运行后输入cmd出现的窗口里面输入java -version看有没有版本的提示？有的话说明你的JDK是对的，那么就是Eclipse的问题了。

    否则你就要设置环境变量信息了：
    依次点击：“我的电脑”－(右键)“属性”－“高级”－“环境变量”，弹出Windows的“环境变量”配置卡。
    在“系统变量”栏下执行三项操作：

    1. 新建“JAVA_HOME”，值：JDK所在的绝对路径(例如D:\Program Files\Java\jdk1.6.0_43)。
    2. 新建“CLASSPATH”(如果已有，则直接编辑)，值：.;%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar(若值中原来有内容，用分号与之隔开)。
    3. 新建“PATH”(如果已有，则直接编辑),值：%JAVA_HOME%\bin\;%JAVA_HOME%\jre\bin;(若值中原来有内容，用分号与之隔开)。

    2.Eclipse无法打开“Failed to load the JNI shared library”

    这是因为64的操作系统安装了32位的JDK(或者是32位的eclipse)导致的，换成64位的即可！

    三、MySQL和Python模块

    1. Linux环境

    建议使用Ubuntu系统（开发是在eclipse上，运行代码则需要在Linux系统上，所以需要Linux环境）

    • 安装mysql和python-mysql模块，命令如下：

      $ apt-get install mysql-server python-mysqldb    \\提示输入mysql密码

    2. Windows环境

    2.1 安装mysql：
    方法一：下载安装包

    • MySQL官网下载安装包mysql_installer_community_V5.6.21.1_setup.1415604646.msi，一步步安装

    方法二：mysql免安装版

    • MySQL官网Download-->Community-->MySQL Community Server，下载 mysql-5.5.47-winx64.zip安装包
    • 解压，将以下载的MySQL压缩包解压到自定义目录下,我的解压目录是："F:\MySQL\mysql-5.5.47-winx64"
    • 将解压目录下默认文件 my-default.ini拷贝一份，改名my.ini（如果没有my-default.ini,可自己新建my.ini或者从其他地方中获取）

    • 添加环境变量，操作如下：

      a)    右键单击我的电脑->属性->高级系统设置(高级)->环境变量
    点击系统变量下的新建按钮
    输入变量名：MYSQL_HOME
    输入变量值：D:\Program Files\mysql-5.6.11-winx64
    #即为mysql的自定义解压目录。
b)    选择系统变量中的Path
    点击编辑按钮
    在变量值中添加变量值：%MYSQL_HOME%\bin
    注意是在原有变量值后面加上这个变量，用;隔开，不能删除原来的变量值，

    • 将mysql注册为windows系统服务

      a)从控制台进入（管理员身份运行）到MySQL解压目录下的 bin 目录下：
b)输入服务安装命令：
    mysqld.exe install MySQL --defaults-file="F:\MySQL\mysql-5.5.47-winx64\my.ini"    \\解压目录下修改的my.ini文件

安装成功后会提示服务安装成功。移除服务命令为：mysqld remove

注1：my.ini文件放在MySQL解压后的根目录下
注2：在windos的cmd下安装mysql，切换到mysql的bin目录下面执行：mysqld --install
报错信息如下：Install/Remove of the Service Denied
解决办法：打开cmd.exe程序的时候选择“用管理员身份打开”。

    • 启动MySQL服务

      方法一：
    启动服务命令为：net start mysql
方法二：
    打开管理工具 服务，找到MySQL服务。
    通过右键选择启动或者直接点击左边的启动来启动服务。

    • 修改root账号的密码（刚安装完成时root账号默认密码为空，此时可以将密码修改为指定的密码。如：123456）

      c:>mysql –uroot
mysql>show databases;
mysql>use mysql;
mysql>UPDATE user SET password=PASSWORD("123456") WHERE user='root';
mysql>FLUSH PRIVILEGES;
mysql>QUIT

    2.2 安装python-mysql模块：MySQL-python-1.2.5.win32-py2.7

    • 前往Python官网–>PyPI–>search MySQL-python，下载MySQL-python-1.2.5.win32-py2.7.exe，下一步下一步安装。

    • 安装MySQL-python-1.2.5.win32-py2.7报错

      报错：Python version 2.7 required,which was not found in the registry.
      解决办法：下载registry.py脚本，把脚本放到任意一盘中执行，运行–>cmd，执行python registry.py
      例：registry.py脚本在F:\MySQL\data目录下

      >f:    \\F盘
>cd MySQL/data \\切换到脚本所在路径
>python registry.py \\执行脚本
python 2.7 is already registered    \\OK

    2.3安装WindowsMySQL数据库管理工具：Navicat_for_MySQL

    • Navicat数据库管理工具，搜索引擎搜索Navicat，下载安装，过程简单，略。

    四、Python的easy_install和pip

    • 为什么要装easy_install?

    easy_install和pip都是用来下载安装Python一个公共资源库PyPI的相关资源包的，那它是干什么的？
    正常情况下，我们要给Python安装第三方的扩展包，我们必须下载压缩包，解压缩到一个目录，然后命令行或者终端打开这个目录，然后执行python setup.py install来进行安装。
    这样很繁琐，如果我们直接命令行执行easy_install 包名就把最新版的包装上去了，是不是很方便。
    所以easy_install就是为了我们安装第三方扩展包更容易而应运而生的。pip同

    • 如何找到easy_install？
      前往Python官网–>PyPI–>搜索ez_setup或setuptools，就能找到它。

    • 首先安装easy_install

      $ wget https://bootstrap.pypa.io/ez_setup.py -O - | sudo python

    • 安装pip：可以通过east_install pip安装

      $ easy_install pip

    pip改善了不少easy_install的缺点，如此说來pip应该是略胜一筹，不过它还不能够完全取代对方，因為目前有很多套件还是得用easy_install安裝

    easy_install：
安裝：easy_install PackageName
更新：easy_install -U PackageName
移除：easy_install -m PackageName
帮助：easy_install --showhelp
pip：
安裝：pip install PackageName
更新：pip install -U PackageName
移除：pip uninstall PackageName
查找：pip search PackageName
帮助：pip help

    五、Windows结合Linux开发环境（samba+映射磁盘）

    Linux运维开发人员要做自动化，很多时候代码是不能在eclipse上执行的，而是要放到Linux系统下运行，而每次运行代码，都需要先登陆服务器，再将代码上传到虚拟机上，然后运行的做法，麻烦而又效率低。

    在windows eclipse上开发，代码Linux下运行的方法：

    1. 需要一个安装好的Ubuntu系统，初始化呢完毕的虚拟机，并安装好samba服务

    samba的安装配置使用搜索引擎可以办到，不多说，贴几行命令：

    $ sudo su -
$ sudo apt-get insall samba
$ sudo apt-get install smbfs
$ sudo mkdir -p /home/share/
$ sudo chmod 777 /home/share/
$ suco cp /etc/samba/smb.conf /etc/samba/smb.conf.bak
$ sudo vim /etc/samba/smb.conf

    在smb.conf最后添加

    [share]
  comment = Ubuntu share
  browseable = yes
  public = yes
  path =  /home/share
  writable = yes
  guest ok = yes

    2. 登录samba目录


    3. 登录进去后，右键点击共享目录(share)，点击映射网络驱动器映射个盘符Z

    4. 现在可以在我的电脑中找到Z盘符

    5. 打开eclipse，会有选择workspace的界面，选中映射的磁盘

    6. 如果没有这个界面，就是之前勾选了Use this the default and do not ask again，在不关闭eclipse前提下，打开第二个eclipse，它会重新让你选择workspace，再选择workspace为映射的盘符，注意这次不要勾选Use this the default and do not ask again了

    7. 最后，在打开eclipse的中，映射python解释器，具体做法在上面文章中给Eclipse安装PyDev插件的第5小步骤配置PyDev插件中涉及到：

    • 在Window菜单中选Preference
    • PyDev->Interpreter - Python，New一个Python解释器，填上解释器名字和路径，路径选相应的python.exe。
