    访问Nginx出现状态码为403 forbidden原因


    Welcome to Fedor’s blog! If you found any problems, you can tell me on GitHub.

    1)nginx配置文件里不配置默认首页参数或者首页文件在站点目录下没有

    2)模拟站点目录下没有配置文件里指定的首页文件index.php index.html index.htm。

    # rm -f ../../html/www/index.html#<==删除物理首页文件
# curl -I -s 10.0.0.8|head-1
HTTP/1.1 403 Forbidden

    3)站点目录或内部的程序文件没有Nginx用户访问权限。

    4)Nginx配置文件中设置allow、deny等权限控制，导致客户端没有没权限访问。
