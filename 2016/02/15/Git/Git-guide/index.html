Git学习备忘

1. 流程及分支

流程

在本地文件夹建立一个名为example的repo，并做一些初始化配置。

$ git init example

在本地文件夹加入或删除文件后，需要用以下命令更新staged area。后者临时保存用户所提交的更改（如 add/rm），以备下一步的commit。

$ git add file1
$ git add .
$ git rm file2

把staged area中所保存的file1的更新信息删除，恢复之前的样子。

$ git reset file1

把上一步 staged area 中所保存的更改进行整理，更新本地的repo（准确的说，是本地repo的master分支），以备后面的push。

$ git commit -m 'Add a new file'
$ git checkout --file1    \\把本地 repo 中 file1的更新还原到之前的状态。

添加服务器端的repo地址，取名为origin（可以随意命名），以备稍后提交本地的repo到这个服务器端的repo。

$　git remote add origin https://[服务器端 repo 地址]

把本地repo的master分支提交到刚才添加的那个服务器端分支origin。-u表示记住后面的参数，以后 push 的时候直接用git push命令即可。

$ git push -u origin master

分支

创建并切换到new-branch分支，删除new-branch分支，这个分支可以用来修复master中代码的bug，然后在merge（合并）到master。

$ git checkout -b new-branch
$ git branch -d new-branch

切换回master分支，然后把刚才新分支(new-branch)对代码所做的更改合并(merge)进来。

$ git checkout master
$ git merge new-branch
$ git push origin master

提交至服务器

$ git commit -a -m “Add first draft of new branch”
$ git push

2. Pull Request在Forking工作流中使用示例

适用于小团队的开发协作和第三方开发者向开源项目的贡献。
示例中，A是个开发，B是项目维护者。各自有一个公开的github仓库，而B的仓库包含了正式工程

A fork正式项目

A先要forkB的githubt仓库，开始项目的开发。A登陆github，浏览到B的仓库页面，点Fork按钮。

A克隆自己的github仓库

A克隆自己刚才fork出来的github仓库，以在本机上准备出工作拷贝。
git clone会自动创建origin远程别名，是指向之前fork出来的仓库。命令如下：

$ git clone https://user@bitbucket.org/user/repo.git

A开发新功能

改代码前，A要为新功能先新建一个新分支,用这个分支作为Pull Request的源分支。
创建并切换分支，命令如下：

$ git checkout -b new-branch

*编辑代码，开发新功能*

提交，命令如下：

$ git add .
$ git add --update .
$ git commit -a -m “Add first draft of some feature”

A push功能到她的github仓库中

A完成了功能后，push功能到她自己的github仓库中（不是正式仓库），用下面简单的命令：

$ git push origin new-branch

A发起Pull Request and B review Pull Request
