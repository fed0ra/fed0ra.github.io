    nginx记录响应与POST请求日志


    Welcome to Fedor’s blog! If you found any problems, you can tell me on GitHub.

    生产环境中需要访问某些api中的日志提供传递的json信息，以便做日志获取处理等操作。

    1.nginx日志配置

    # vim /etc/nginx/nginx.conf
...
log_format  mytest  '$remote_addr - $remote_user [$time_local] "$request" '
                  '$request_body $status $body_bytes_sent "$http_referer" '
                  '"$http_user_agent" "$http_x_forwarded_for"';
...

    $request_body变量由nginx自身提供，用于记录POST请求日志

    2.nginx虚拟主机配置

    server {
            listen                80;
            server_name  xx.oo.com;
            charset             utf-8;
            proxy_redirect          off;
            access_log  logs/xx_access.log  mytest;
        ...省略些许配置...

    }

    配置完毕之后，所有/下的请求都会记录到xx_access.log中，包含request日志

    3.日志结果

    日志中包含post请求传递的json串，即成功。
    通过根据此日志，可以及早发现问题并且修复，或者进行相应的处理。

    POST里面的一些内容都是\x22\xE9\x93\xB6\xE8，无法理解
