    一、Zabbix agent两种运行方式

    1、被动模式和主动模式

    zabbix agent的运行方式有两种，主动和被动模式，默认情况下zabbix agent运行在被动模式，也就是默认监听一个10050端口，server端会主动找agent要数据，在这种模式下如果agent太多，那么zabbix server会很繁忙。

    所以zabbix server还有一个主动模式（主动模式参数参考上篇文章结尾Zabbix 3.0安装部署），zabbix agent会主动请求zabbix server，agent会主动向server要它需要干的任务，再定期把数据发送给server，这时候server处于被动接受状态，减少server的压力。

    2、开启主动模式

    打开zabbix web管理界面：
    克隆所需要的模板，默认模板的监控项都是Zabbix agent这种被动模式，需要改成Zabbix agent (active)

    其次修改agent的配置文件：

    Server=127.0.0.1    #注释掉Server
StartAgents=0        #开启zabbix agent主动模式不需监听任何tcp端口，此项参数设置为0
ServerActive=127.0.0.1    #zabbix server端的ip
Hostname=Zabbix server    #zabbix server的主机名

    重启zabbix agent，在web管理界面创建主机，连接克隆后的模板。
    需要注意，创建主机的时候，agent代理接口，IP地址填入0.0.0.0 ，端口填入0 保存

    二、Zabbix proxy

    1、zabbix分布式监控介绍

    在大规模监控情况下，仅仅将agent改成主动模式是不够的，zabbix也支持分布式监控(zabbix proxy)，例如有多个机房，可以把每个机房个部署一个zabbix proxy。
    zabbix proxy的工作原理，其代替zabbix server来收集数据，收集后在本地暂存，然后马上发给zabbix server，这样可以减轻zabbix server的压力，也可以在网络情况比较复杂的情况下部署。

    注意：zabbix proxy只是一个数据的收集器，它不会计算触发器，处理事件，发送警报

    Zabbix proxy is a data collector. It does not calculate triggers, process events or send alerts. For an overview of what proxy functionality is, review the following table:

    2、zabbix proxy部署

    安装zabbix proxy

    # yum install -y zabbix-proxy zabbix-proxy-mysql mysql-server
#zabbix proxy需要数据库，收集完的数据要放到数据库，无web界面

# rpm -ql zabbix-proxy
#查看zabbix-proxy安装了什么东西

# rpm -ql zabbix-proxy-mysql
#查看zabbix-proxy-mysql安装的文件
#需要注意的是，安装zabbix server的时候向数据库导入了3个表，而zabbix-proxy-msyql只用导入表结构这一个文件就可以了

    登录数据库，创建mysql_proxy库并授权

    mysql> create database zabbix_proxy character set utf8;
mysql> grant all on zabbix_proxy.* to zabbix@localhost identified by 'zabbix';
mysql> use zabbix_proxy;
mysql> source xxx/schema.sql;    #sql文件地址通过rpm -ql zabbix-proxy-mysql查看
mysql> exit;

    修改zabbix proxy配置文件/etc/zabbix/zabbix_proxy.conf

    • ProxyMode=0，0主动，1被动，默认0主动模式，主动被动也是相对于zabbix proxy来说的
    • Server=xx.xx.xx.xx，zabbix server的ip，主动模式需填写
    • Hostname=xxx，一定要改，不改回报错
    • DBName=zabbix_proxy，数据库名称
    • DBUser=zabbix，用户名
    • DBPassword=zabbix，密码
    • DBSocket=/var/lib/mysql/mysql.sock，使用sock连接

    在zabbix server上加上代理

    登录zabbix web管理页面，Administration–>proxies–>Create proxy

    • Proxy name，生产上命名最好根据idc或者项目
    • Proxy mode，默认主动
    • Description，描述方便管理

    创建主机的时候唯一注意的是，Monitored by proxy要选择代理的主机进行监控

    agent的ServerActive一定要修改成proxy的IP地址

    三、Zabbix自动化监控

    大规模的监控环境下，成百上千台机器如果手动添加工作量大且无意义，这样就用到了Zabbix自动化监控。
    Zabbix自动化监控有三种方式：

    1、zabbix agent自动注册，agent主动去server端注册。

    • 创建action：web管理页面-->Configuration-->Actions，注意Event source也就是事件来源一定要选择Auto registration，之后点击Create action。
    • name：Agent自动发现，自定义；条件选择：Host metadata like Linux；操作根据需求：Add host,Add to host groups:xx,Link to templates:xx。【这里的意思是当主机元数据（即system.uname获取到的值中）有Linux，就将执行三个动作，添加主机、添加主机组并link一个模板。】
    • 配置文件添加一行配置：HostMetadataItem=system.uname

    2、自动发现，在server主动扫描发现。

    • 创建自动发现：web管理页面-->Configuration-->Discovered，填写自动发现ip地址段及相关信息。
    • 创建自动发现后再创建action：web管理页面-->Configuration-->Actions，，注意Event source也就是事件来源一定要选择Discovery，之后点击Create action。
    • name：Auto discovery Linux server，自定义；条件选择：Received value like Linux,Discovery status=Up,Service type=Zabbix agent，操作根据需求：Add host,Add to host groups:xx,Link to templates:xx。**【意思是当接收到的值为Linux，并且自动发现状态为开启，服务类型是zabbix agent的时候，就将执行三个动作，添加主机、添加主机组并link一个模板。】

    3、使用Zabbix API，server端主动添加。

    Zabbix 3.0安装部署中，安装zabbix server的时候用的是nginx来访问zabbix web，拷贝了PHP网页文件到nginx一个自定义的路径下，找到此路径。

    # cat /usr/local/nginx/conf/extra/zabbix.conf
...
root /data/web/zabbix;
...

# ll /data/web/zabbix/api_jsonrpc.php

    通过查看nginx配置文件找到网页文件存放在/data/web/zabbix下，此路径下有一个文件api_jsonrpc.php，所有的请求都发送到这里。

    Zabbix API具体使用参考官方资料

    使用Zabbix API：

    使用API首先需要验证，发送一个POST数据，数据中附带用户名密码，它会返回一个session id，后面再请求的时候带上这个session id。

    获取验证的例子：

    #使用curl构造一个请求，-s静默，-H文档的类型，-d要post的数据
#用python处理一下返回的数据，便于观看

curl -s -X PORT -H 'Content-Type:application/json' -d '
{
    "jsonrpc": "2.0",
    "method": "user.login",
    "params": {
        "user": "Admin",
        "password": "shifeng1127"
    },
    "id": 1
}' http://xx.xx.xx.xx/zabbix/api_jsonrpc.php | python -mjson.tool

#记下result返回的session id

    获取所有监控的主机id例子：

    curl -s -X PORT -H 'Content-Type:application/json' -d '
{
    "jsonrpc": "2.0",
    "method": "host.get",
    "params": {
        "output": ["hostid"]
    },
    "auth": "0364d3a83f8e1da69c84e9cfd45f3f1a",
    "id": 2

}' http://xx.xx.xx.xx/zabbix/api_jsonrpc.php | python -mjson.tool

    看主机名称，不输出hostid，只让输出host即可。

    加一台主机，并link一个模板例子：

    #1.验证
curl -s -X PORT -H 'Content-Type:application/json' -d '
{
    "jsonrpc": "2.0",
    "method": "user.login",
    "params": {
        "user": "Admin",
        "password": "shifeng1127"
    },
    "id": 1
}' http://xx.xx.xx.xx/zabbix/api_jsonrpc.php | python -mjson.tool

#得到session id：ff537f2268aac8d020a6d854e30d6958
#2.请求API，附带上sessionID
curl -s -X PORT -H 'Content-Type:application/json' -d '
{
    "jsonrpc": "2.0",
    "method": "host.create",
    "params": {
        "host": "app01",
        "interfaces": [
            {
                "type": 1,
                "main": 1,
                "useip": 1,
                "ip": "10.25.106.196",
                "dns": "",
                "port": "10050"
            }
        ],
        "groups": [
            {
                "groupid": "8"
            }
        ],
        "templates": [
            {
                "templateid": "10114"
            }
        ]
    },
    "auth": "ff537f2268aac8d020a6d854e30d6958",
    "id": 1
}' http://xx.xx.xx.xx/zabbix/api_jsonrpc.php | python -mjson.tool

#修改ip、groupid和templateid
