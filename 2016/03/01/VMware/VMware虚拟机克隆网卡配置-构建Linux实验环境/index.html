    一、创建虚拟机并安装操作系统

    1.新建虚拟机

    自定义新建一个虚拟机
    –>选择虚拟机兼容性（默认即可）
    –>按住客户机操作系统（我们选稍后安装操作系统）
    –>选择客户机操作系统（使Linux系统，选择Linux，版本根据镜像，镜像是centos 6 64位，则选择CentOS 64位）
    –>命令虚拟机，自定义，位置存放在一个统一路径下，（F:\Virtual Machines\CentOS-Server-01）如图：

    –>处理器配置，自定义
    –>虚拟机内存，根据物理机和情况自定义（512M）
    –>网络类型（暂时默认，后为配置）
    –>I/O控制器类型和磁盘类型自定义，也可默认
    –>选择磁盘（创建新虚拟磁盘）
    –>指定磁盘容量（10G，用多少占用多少，不会分配10G后立刻占用硬盘10G空间的）
    –>指定磁盘文件默认下一步，完成创建虚拟机

    二、编辑虚拟机配置

    4.物理机共享网络配置

    在网络管理中，右键物理网卡–>属性–>共享，勾选允许其他网络用户通过此计算机的Internet连接来连接，并选择VMnet8，即以太网3，之后会给VMnet8分配一个IP地址，这里分配的是192.168.26.2，记住之后配置虚拟机上网时会用此IP作为网关

    5.在VMware工具栏中点击编辑–>虚拟网络编辑

    6.打开虚拟网络编辑器后，会出现多块虚拟网卡相关信息，找到VMnet1和VMnet8，修改VMnet8的网段为你物理机网络连接中VMnet8的网段，即192.168.26.0，DHCP服务视情况开启否。NET模式，有一个NET设置，点开编辑网关为本机VMnet8的ip，即26.2。


    三、初始化系统配置网卡

    1.网卡配置

    备份eth0网卡，修改其配置文件：

    # cp /etc/sysconfig/network-scripts/ifcfg-eth0{,.bak}
# cd /etc/sysconfig/network-scripts/
# cat ifcfg-eth0
DEVICE=eth0
TYPE=Ethernet
ONBOOT=yes
NM_CONTROLLED=yes
BOOTPROTO=none
IPADDR=192.168.174.10
NETMASK=255.255.255.0

    IP地址使用仅主机模式网段

    拷贝eth0，命名为eth1，编辑eth1

    # cp ifcfg-eth0 ifcfg-eth1
# cat ifcfg-eth1
DEVICE=eth1
TYPE=Ethernet
ONBOOT=yes
NM_CONTROLLED=yes
BOOTPROTO=none
IPADDR=192.168.26.10
GATEWAY=192.168.26.2
NETMASK=255.255.255.0

    IP地址使用NAT模式，即VMnet8网段，网关设置成物理机VMnet8的IP地址，即192.168.26.2。

    注：为了克隆虚拟机后网络没有问题，去掉网卡配置中的UUID

    重启网卡：

    # /etc/init.d/network restart
# ping g.cn

    Ubuntu网卡设置如下：

    # The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
auto eth0
iface eth0 inet static
address 192.168.174.20
netmask 255.255.255.0

auto eth1
iface eth1 inet static
address 192.168.26.20
gateway 192.168.26.2
netmask 255.255.255.0

    2.设置主机名

    # cat /etc/sysconfig/network
NETWORKING=yes
HOSTNAME=CentOS-Server01.example.com

    3.设置主机名让它可以解析，长短域名都可以解析

    # cat /etc/hosts
192.168.174.10 CentOS-Server01 CentOS-Server01.example.com

    4.关闭防火墙和selinux

    # /etc/init.d/iptables stop
# chkconfig iptables off
# sed -i 's@SELINUX=enforcing@SELINUX=disabled@' /etc/selinux/config

    5.安装基础工具

    # yum install screen tree lrzsz gcc glibc gcc-c++ make pathch openssl-devel
# yum update -y && reboot

    返回目录

    四、克隆虚拟机

    1.删除udev下的文件：

    # rm -rf /etc/udev/rules.d/70-persistent-net.rules

    2.关闭客户机

    初始化好后最好先做个快照。
    右键点击客户机–>管理–>克隆，弹出克隆虚拟机使用向导，下一步
    –>选择克隆源，可以使用当前状态，也可用做好快照来进行克隆
    –>创建完整克隆

    至此完毕，以后需要直接从快照克隆一台新的虚拟机，修改IP地址即可。

    返回目录

    五、问题解决

    如果确定以上操作无问题，但ping不通网关，可能是windows上的VMware NET Service服务没有启动。
