    Zabbix 3.0安装部署


    Welcome to Fedor’s blog! If you find any problems, you can tell me in any way.

    二、Zabbix Server的安装

    Zabbix简介

    1、zabbix是一个基于WEB界面的，并提供分布式系统监视以及网络监视功能的企业级的开源解决方案。 zabbix能监视各种网络参数，保证服务器系统的安全运营；并提供柔软的通知机制以让运维人员能够快速定位。

    2、zabbix通过C/S模型来收集数据，通过B/S模型模式在web端展示和进行配置。

    • 被监控端：主机设备通过安装agent收集数据并发送给server端，网络设备通过SNMP收集数据并发送给server端。
    • server端：通过收集SNMP和agent发送来的数据，将数据写入到后台数据库中（例如：mysql，oracle），在通过web展示出来。

    3、运行环境

    • zabbix server 需要运行在LAMP或者是LNMP的环境下；
    • agent可以部署在目前常见的OS中，例如linux，windows，HPUX，Solaris，Sun；
    • SNMP可以支持常见的网络设备。

    一、Zabbix的部署准备

    系统：CentOS release 6.5

    软件版本：nginx 1.7.0、PHP 5.6.25、mysql-5.5.36，zabbix server 3.0.4

    1、环境准备与配置

    需要提前安装好nginx/php/mysql，安装略。
    php装好后需修改几项参数，如下：

    sed -i "s/;date.timezone =/date.timezone = Asia\/Shanghai/g" /usr/local/php/etc/php.ini
sed -i "s#max_execution_time = 30#max_execution_time = 300#g" /usr/local/php/etc/php.ini
sed -i "s#post_max_size = 8M#post_max_size = 16M#g" /usr/local/php/etc/php.ini
sed -i "s#max_input_time = 60#max_input_time = 300#g" /usr/local/php/etc/php.ini
#下面另个参数php5.6默认开启了
#memory_limit = 128M
#upload_max_filesize = 2M

    说明：

    • 1、mysql是zabbix数据存放主机，nginx和php用于zabbix的web前端展示主机（nginx/php/mysql编译或yum安装均可）；
    • 2、zabbix服务端装在监控主机上(服务端最好编译安装，以提高监控机性能)；
    • 3、agentd是装在被监控主机上（yum安装方便快捷）。
    • 4、通常情况下这四个角色可以搭建在同一个主机上，也可以分开部署了。

    二、Zabbix Server的安装

    1、zabbix源文件的下载与编译安装

    # yum install unixODBC-devel net-snmp-devel libssh2-devel OpenIPMI-devel openldap-devel -y
# wget http://120.52.73.47/downloads.sourceforge.net/project/zabbix/ZABBIX%20Latest%20Stable/3.0.4/zabbix-3.0.4.tar.gz
# tar xf zabbix-3.0.4.tar.gz 
# cd zabbix-3.0.4
# ./configure --prefix=/usr/local/zabbix-3.0.4 --enable-server --enable-agent --with-mysql --enable-ipv6 --with-net-snmp --with-libcurl --with-libxml2 --enable-proxy --with-ssh2 --with-openipmi --with-unixodbc --with-ldap
# make install

    通过命令cat /etc/services | grep zabbix检查zabbix是否被添加进入系统服务文件中，如果看到如下信息则代表已经添加成功，或者可以自行手动添加到/etc/services文件中：

    zabbix-agent 10050/tcp     #Zabbix Agent
zabbix-agent 10050/udp    #Zabbix Agent
zabbix-trapper 10051/tcp   #Zabbix Trapper
zabbix-trapper 10051/udp  #Zabbix Trappe

    2、创建zabbix用户

    # groupadd zabbix
# useradd -s /sbin/nologin -g zabbix -M zabbix

    3、建立zabbix所需的数据库

    添加mysql数据库的zabbix用户，并设置密码：先用root身份登入mysql中，并执行以下命令：

    mysql> create database zabbix character set utf8 collate utf8_bin;
mysql> grant all privileges on zabbix.* to zabbix@localhost identified by '<password>';
mysql> flush privileges;
mysql> quit;

    导入zabbix数据库（确保路径在zabbix源码路径下）而后在下载的源文件中存在一个database/mysql子目录，中间存放在建立zabbix数据库结构的SQL脚本文件。
    先进入到zabbix源文件所在的目录，并执行以下语句：

    # cd database/mysql
# mysql -u zabbix -p <password> zabbix < schema.sql
# mysql -u zabbix -p <password> zabbix < images.sql
# mysql -u zabbix -p <password> zabbix < data.sql

    4、创建zabbix的的日志文件和连接

    # mkdir /var/log/zabbix
# chown zabbix.zabbix /var/log/zabbix
# ln -s /usr/local/zabbix-3.0.4/etc /etc/zabbix
# ln -s /usr/local/zabbix-3.0.4/bin/* /usr/bin/
# ln -s /usr/local/zabbix-3.0.4/sbin/* /usr/sbin/

    5、拷贝与配置zabbix启动文件

    # cp misc/init.d/fedora/core/zabbix_* /etc/init.d/
# chmod 755 /etc/init.d/zabbix_*
# sed -i "s#BASEDIR=/usr/local#BASEDIR=/usr/local/zabbix-3.0.4#g" /etc/init.d/zabbix_server
# sed -i "s#BASEDIR=/usr/local#BASEDIR=/usr/local/zabbix-3.0.4#g" /etc/init.d/zabbix_agentd

    6、配置zabbix_server.conf服务端文件

    编辑并修改/etc/zabbix/zabbix_server.conf文件，并修改如下条目：

    • DBHost=xx.xx.xx.xx 数据库的地址
    • DBName=zabbix zabbix数据库名称
    • DBUser=zabbix 连接zabbix库的用户
    • DBPassword=zabbix 数据库密码（这里为了演示，先前设置了zabbix数据库密码为）
    • LogFile=/var/log/zabbix/zabbix_server.log 日志

    7、拷贝PHP网页文件到nginx中

    将前端文件拷贝到项目目录下：

    # cp -r ./frontends/php/ /data/web/zabbix

    编辑nginx虚拟主机：

    # cd /usr/local/nginx/conf/
# mkdir extra/^C
# cd extra/
# vim zabbix.conf

    内容如下：

    server {
        listen 80;
        server_name xx.xx.xx.xx;
        access_log logs/xx.xx.xx.xx.log main;
        index index.html index.php index.html;
        root /data/web/zabbix;

        location / {
                try_files $uri $uri/ /index.php?$args;
        }

        location ~ ^(.+.php)(.*)$ {
                fastcgi_split_path_info ^(.+.php)(.*)$;
                include fastcgi.conf;
                fastcgi_pass 127.0.0.1:9000;                fastcgi_index index.php;
                fastcgi_param PATH_INFO $fastcgi_path_info;
        }

}

    编辑nginx.conf配置文件，在http标签中加入：

    include extra/*.conf;

    8、设置zabbix开机自启并启动

    chkconfig zabbix_server on
chkconfig zabbix_agentd on
/etc/init.d/zabbix_server start
/etc/init.d/zabbix_agentd start

    返回目录

    三、通过Web设置zabbix

    先确保mysql/nginx/php服务均已启动。在浏览器中输入地址：http://xx.xx.xx.xx（nginx配置文件中配置的ip地址或域名），则可看到初始化欢迎界面。

    PS：
    1、初始的账号名为：Admin，初始密码为：zabbix；
    2、检测那些配置不显示ok，按照提示修改即可；
    3、最后一步生成不了配置文件，按照提示将配置文件下载下来，放到相关目录即可。

    返回目录

    四、客户端agentd安装

    1、装yum源

    # rpm -ivh http://repo.zabbix.com/zabbix/3.0/rhel/6/x86_64/zabbix-release-3.0-1.el6.noarch.rpm

    2、安装

    # yum install zabbix-agent -y

    3、修改配置文件

    /etc/zabbix/zabbix_agentd.conf是客户端配置文件，该文件需要修改如下条目：
    1、Server=127.0.0.1，zabbix server服务端的ip。
    2、StartAgents=0，开启zabbix agent主动模式不需监听任何tcp端口，此项参数设置为0。
    3、ServerActive=127.0.0.1，zabbix server端的ip，主动模式需此项参数。
    4、Hostname=Zabbix server，zabbix server的主机名，主动模式需要。
    5、UnsafeUserParameters=0，默认是不启用自定义脚本功能的，要自定义key，需开启，设置为1。

    开启zabbix agent的active主动模式：需注释掉掉1，开启第2项将参数为0，修改3项zabbix server ip地址，修改第4项zabbix server的主机名，如果需要自定义key需开启第5项。

    4、设置开机自己并启动zabbix-agent

    # chkconfig zabbix-agent on
# /etc/init.d/zabbix-agent start

    返回目录
