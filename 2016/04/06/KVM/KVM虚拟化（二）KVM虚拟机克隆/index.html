KVM虚拟化（二）KVM虚拟机克隆


Welcome to Fedor’s blog! If you find any problems, you can tell me in any way.

一、复制配置文件与磁盘文件克隆
二、KVM主机本机虚拟机直接克隆

本文克隆虚拟机为CentOS release 6.6 X64。
kvm虚拟机的克隆分为两种情况。

  • 复制配置文件与磁盘文件克隆(异机的静态迁移)
  • KVM主机本机虚拟机直接克隆

一、复制配置文件与磁盘文件克隆

(1)导入kvm虚拟机配置文件

# virsh list --all
# virsh dumpxml centos2 > /etc/libvirt/qemu/centos-clone.xml
# ll /etc/libvirt/qemu

将centos2虚拟机的配置文件导出重命名为centos-clone.xml

(2)复制KVM虚拟机磁盘文件

# cp /data/img/kvm_centos2.img /data/img/kvm_centos-clone.img
# ll /data/img/

(3)编辑修改配置文件

修改name,uuid,disk位置,mac,vnc端口，uuid和mac随意修改

# cat /etc/libvirt/qemu/centos-clone.xml
<name>centos-clone</name>
<uuid>62492255-296b-80e4-5431-ae6795040437</uuid>
<source file=’/data/img/kvm_centos-clone.img’/>
<mac address='52:54:00:0a:bd:32'/>
<graphics type=’vnc’ port=’5916′ autoport=’no’ listen=’0.0.0.0′>

name改成新虚拟机名字centos-clone
uuid随意修改一个字符，会自动生成
disk位置，克隆的新的虚拟机的磁盘位置
mac同uuid

(4)定义新虚拟机配置文件

# virsh define /etc/libvirt/qemu/centos-clone.xml
#再查看，克隆的虚拟机已经存在，处于关闭状态
# virsh list --all
#启动虚拟机
# virsh start centos-clone

(5)VNC使用IP:5916端口，登录虚拟机进行主机名，IP等修改

#修改主机名
# hostname centos-clone
# vi /etc/sysconfig/network
HOSTNAME=centos-clone

#修改IP
# vi /etc/sysconfig/network-scripts/ifcfg-eth0

注：重启网卡错误解决方法

# vi /etc/sysconfig/network-scripts/ifcfg-eth0
#删除虚拟机eth0中的UUID和HWADDR
#并且删除udev下
# rm -rf /etc/udev/rules.d/70-persistent-net.rules
# reboot

返回目录

二、KVM主机本机虚拟机直接克隆

(1)查看虚拟机，寻找一台关闭状态的虚拟机作为克隆的源

# virsh list --all

这里以centos2作为源，创建名为centos-clo的虚拟机，使用/data/img/kvm_ctest-clo.img为磁盘文件

(2)克隆

# virt-clone -o centos2 -n centos-clo -f /data/img/kvm_centos-clo.img
#克隆成功，查看是否已经生成xml和磁盘文件
# ll /etc/libvirt/qemu
# ll /data/img/

(3)启动虚拟机

# virsh list --all
# virsh start centos-clo

(4)登录虚拟机进行主机名，IP等修改

#修改主机名
# hostname centos-clo
# vi /etc/sysconfig/network
HOSTNAME=centos-clo

#修改IP
# vi /etc/sysconfig/network-scripts/ifcfg-eth0
