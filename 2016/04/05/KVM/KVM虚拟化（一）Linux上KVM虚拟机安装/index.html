    KVM虚拟化（一）Linux上KVM虚拟机安装


    Welcome to Fedor’s blog! If you found any problems, you can tell me on GitHub.

    一、虚拟化环境准备
    二、KVM虚拟机安装
    三、KVM联网与配置iptables端口转发

    一、虚拟化环境准备

    1、修改内核模式为兼容内核启动

    #查看系统版本，
# cat /etc/redhat-release
#查看内核版本，内核版本在2.6.20以下需要升级。在2.6.20以后KVM成为Linux的一个内核模块
# uname -a

    2、关闭selinux和防火墙

    # /etc/init.d/iptables stop
# chkconfig iptables off
# cp /etc/selinux/config /etc/selinux/config.`date +"%Y-%m-%d_%H:%M:%S"`
# sed -i 's@SELINUX=enforcing@SELINUX=disabled@' /etc/selinux/config
# setenforce 0
# reboot

    3、虚拟化环境安装

    (1)查看是否支持虚拟化。半虚拟化是不能运行与安装KVM虚拟机的。若实际环境没有, 需要到主板BIOS中开启Virtual Technolege(VT, 虚拟化技术)。使用命令查看，含有红色vmx字样则支持虚拟化。

    # egrep '(vmx|svm)' --color=always /proc/cpuinfo

    注：interl的VT技术在cpu的功能标签里叫vmx，AMD的AMD-V技术叫svm

    (2)配置yum

    # rpm -ivh http://mirrors.ustc.edu.cn/fedora/epel//6/x86_64/epel-release-6-8.noarch.rpm

    (3)安装kvm软件包和kvm虚拟化一些管理工具包(libguestfs-tools)

    #yum -y install kvm qemu-kvm python-virtinst  virt-manager qemu-kvm-tools libvirt libvirt-python tunctl bridge-utils virt-viewer virt-v2v libguestfs-tools gcc* numactl-devel

    注：安装qemu-kvm会加载kvm模块，cat /etc/sysconfig/modules/kvm.modules中代码有写到。

    4、查看虚拟化环境

    (1)查看kvm模块支持

    # lsmod |grep kvm
kvm_intel              55624  0
kvm                   338535  1 kvm_intel

    (2)启动libvirtd，查看虚拟机环境

    # /etc/init.d/libvirtd start
# virsh list --all
Id    Name                           State

    (3)查看虚拟工具版本

    # virsh –version
# virt-install --version
# ln -s /usr/libexec/qemu-kvm /usr/bin/qemu-kvm
# qemu-kvm -version

    启动libvirtd报错

    # /etc/init.d/libvirtd start
启动 libvirtd 守护进程：libvirtd: relocation error: libvirtd: symbol numa_nodes_ptr, version libnuma_1.2 not defined in file libnuma.so.1 with link time reference
#解决办法
#yum install -y numactl-devel

    返回目录

    二、KVM虚拟机安装

    1、上传ISO文件，CentOS-6.6-x86_64-bin-DVD1.iso

    # ll /data/CentOS-6.6-x86_64-bin-DVD1.iso

    2、开始安装Centos6.6 X64

    (1)raw格式磁盘

    使用qemu-img创建kvm硬盘，raw代表裸硬盘，裸设备文件系统特性是分多大占用多大

    # qemu-img create -f raw /data/img/kvm_ctest2.img 5G

    查看磁盘信息，以后所有磁盘的操作都用qemu-img进行操作

    # cd /date/img
# ls -lh    #查看大小
# qemu-img info /data/img/kvm_ctest2.img

    用KVM创建虚拟机

    # virt-install --virt-type kvm --name centos2 --ram 512 --vcpus=1 --cdrom=/data/CentOS-6.6-x86_64-bin-DVD1.iso --network network=default --graphics vnc,port=5912,listen=0.0.0.0 --noautoconsole --os-type=linux --os-variant=rhel6 --disk path=/data/img/kvm_centos2.img

    也可以将命令写入脚本，更直观，然后执行脚本创建虚拟机

    # cat create_kvm.sh
virt-install \
--virt-type kvm \
--name centos2 \
--ram 512 \
--vcpus=1 \
--cdrom=/data/CentOS-6.6-x86_64-bin-DVD1.iso \
--network network=default \
--graphics vnc,port=5912,listen=0.0.0.0 --noautoconsole --os-type=linux \
--os-variant=rhel6 \
--disk path=/data/img/kvm_centos2.img

# sh create_kvm.sh
# echo $?   #0
# virsh list --all

    注:每行”\”后面不要带有空格。
    参数说明:
    –name指定虚拟机名称
    –ram分配内存大小（默认大小MB）
    –vcpus分配CPU核心数，最大与实体机CPU核心数相同
    –disk指定虚拟机镜像，size指定分配大小单位为G。
    –network网络类型，一般用的应该是bridge桥接。
    –accelerate加速
    –cdrom指定安装镜像iso
    –vnc启用VNC远程管理，一般安装系统都要启用。
    –vncport指定VNC监控端口，默认端口为5900，端口不能重复。
    –vnclisten指定VNC绑定IP，默认绑定127.0.0.1，这里改为0.0.0.0。
    –os-type=linux,windows
    –os-variant=

    raw格式磁盘转换成qcow2格式：

    虚拟机处于关闭状态

    # cd /data/img/
# qemu-img convert -c -f raw -O qcow2 kvm_ctest2.img kvm_ctest2-qcow2.img
# ll -lh        #查看新格式大小

    -f 源镜像的格式
    -O 目标镜像的格式

    修改xml文件，然后启动

    # cd /etc/libvirt/qemu
# virsh edit centos2    #不带后缀`.xml`，直接vim修改`xml`文件不生效的，必须用`virsh edit`
Domain centos2 XML configuration edited.

<driver name='qemu' type='raw' cache='none'/>
<source file='/data/img/kvm_ctest2.img'/>改成
<driver name='qemu' type='qcow2' cache='none'/>
<source file='/data/img/kvm_ctest2-qcow2.img'/>
# virsh list --all
# virsh destroy centos2    #修改xml后reboot不生效，必须关闭
# virsh start centos2

    查看进程，看是否用的是修改过的磁盘

    # ps -ef | grep kvm

    (2)qcow2格式(空间动态增长)，生产环境建议用qcow2，裸设备(即raw)出问题不好处理

    创建qcow2格式虚拟磁盘

    # qemu-img create -f qcow2 /data/img/centos3.img 3G
# echo $?

    用KVM创建虚拟机

    # cat create_qcow_kvm.sh
virt-install \
--virt-type kvm \
--name centos3 \
--ram 512 \
--vcpus=1 \
--cdrom=/data/CentOS-6.6-x86_64-bin-DVD1.iso \
--network network=default \
--accelerate \
--graphics vnc,port=5913,listen=0.0.0.0 --noautoconsole --os-type=linux \
--os-variant=rhel6 \
--disk path=/data/img/centos3.img,format=qcow2

# sh create_qcow_kvm.sh
# echo $?
# virsh list --all

    (3)开始通过VNC进行连接，进行安装过程

    IP:5912
    IP:5913
    安装步骤过程略

    注：连接VNC自然一闪而过解决办法：
    设置VNC，vnc-options，expert-colourlevel改为full即可。

    安装完成后，登录主机修改主机名和IP
    注意：以下需要在kvm虚拟机中操作

    #修改主机名
# hostname centos2
# vi /etc/sysconfig/network
HOSTNAME=centos2

#修改IP，自动获取IP
# vi /etc/sysconfig/network-scripts/ifcfg-eth0
# /etc/init.d/network restart
# ip a

    清空iptables规则, 通过sshd登录该主机

    # iptables -F
# iptables -X
# /etc/init.d/iptables restart

    (4)安装完成后，可以登录操作系统查看

    查看虚拟机列表。安装完操作系统自动关闭, 需要手动启动

    # virsh start centos2 centos3 started
# virsh list --all
# ps qux | grep kvm

    修改内存，可通过以下命令，虽然有修改内存的参数但不可以修改。

    # virsh setmem centos2 512

    (5)安装电源服务

    # yum -y install acpid
# /etc/init.d/acpid start
# chkconfig acpid on

    至此Centos 6.6 X64KVM虚拟机安装完毕

    返回目录

    三、KVM联网与配置iptables端口转发

    上述操作安装好VM后，VM通过NAT网络获取的IP地址无法与其他网段的服务器进行通信。

    1、KVM联网问题

    切换到KVM服务器，编辑文件开启KVM服务器的IP转发功能。

    # vim /etc/stsctl.conf

    把其中的net.ipv4.ip_forward = 0修为net.ipv4.ip_forward = 1

    # sysctl –p

    使用以下命令开启KVM服务器的IPtables的转发功能。

    # iptables -t nat -A POSTROUTING -o em1 -j MASQUERADE

    em1根据情况修改，通过em1联网，一般为相当于eth1，这里是em1。
    尝试使用virbr0（自动生成的网卡）但连不上网。

    最后，切换到虚拟机中ping一下查看网络情况，此时虚拟机可以与服务器外网正常通信。

    2、为VM配置iptables端口转发

    因为有些业务是在公网的，有时候为了资金考虑不得不尽量节省公网IP的个数。比如现在想管理KVM中的VM，通过SSH方式。但是KVM服务器对外只有一个公网IP，而且KVM服务器是在IDC机房中。
    如果要达到上述的要求，只能在KVM服务器为VM配置IPtables端口转发。

    1、通过iptables端口转发实现ssh远程登录虚拟机

    以上述VM为例，目前该KVM的公网IP为192.168.2.99，VM的IP为192.168.122.11，现在我要求通过访问KVM的8011端口访问VM的22端口。

    设置如下IPtables规则：

    # iptables -t nat -A PREROUTING -d 192.168.2.99 -p tcp -m tcp --dport 8011 -j DNAT --to-destination 192.168.122.11:22

    把所有访问192.168.2.99:8011的请求转发到192.168.122.11:22的端口上。

    # iptables -t nat -A POSTROUTING -s 192.168.122.0/255.255.255.0 -d 192.168.122.11 -p tcp -m tcp --dport 22 -j SNAT --to-source 192.168.122.1

    把所有来自192.168.122.0/255.255.255.0网段访问192.168.122.11:22的数据全部通过192.168.122.1这个网关转发出去。

    # ssh -p 8011 root@192.168.2.99

    远程登录后使用ifconfig命令查看ip是否是VM的ip192.168.122.11

    2、通过iptables端口转发实现访问虚拟机的web网站

    在虚拟机192.168.122.11中搭建web网站，通过KVM的IPtables端口转发下，公网就可以访问了。

    # iptables -t nat -A PREROUTING -d 192.168.2.99 -p tcp -m tcp --dport 8020 -j DNAT --to-destination 192.168.122.11:80
# iptables -t nat -A POSTROUTING -s 192.168.122.0/255.255.255.0 -d 192.168.122.11 -p tcp -m tcp --dport 80 -j SNAT --to-source 192.168.122.1

    浏览器访问192.168.2.99:8020，就可以访问到虚拟机192.168.122.11的web服务器。

    # /etc/init.d/iptables save
# more /etc/sysconfig/iptables

    保存防火墙规则，使重启KVM服务器后防火墙依然生效。

    3、其他

    iptables做端口转发后生效问题？
    一般来讲，iptables做端口转发后可以立即生效，但取决于内网IP是否更换觉得，意思就是如果你内网更换IP后，DNS将同步所以转发的IP地址，此时需要同步时间，如果你的DNS设置一个小时，那么你在做IPTABLES后可能会在1个小时以后可以访问，如果你的DNS大于1小时，当然你的iptables生效可能会在1个小时以后，仅供参考

    linux iptables端口转发问题
    内网用户访问外网的web服务器的80端口都转换为内网代理服务器squid的默认端口3128。只要代理服务器能够访问互联网，内网用户也就可以访问互联网。格式：

    # iptables –t nat –A PREROUTING –s 子网地址/子网掩码 -p tcp --dport 80 -j REDIRECT – –to-posts 3128

    参考文章：http://www.bkjia.com/Linuxjc/877147.html
