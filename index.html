Zabbix自定义报警


Welcome to Fedor’s blog! If you find any problems, you can tell me in any way.

Zabbix在配置好触发器后，需要有触发的动作(actions)才会报警

一、配置

1. 发送短信示例

由于发送短信有字符限制，在创建的时候尽量精简信息内容，以保证准确收到信息。

步骤：配置–>动作–>事件源（触发器）–>创建动作

动作：

名称：Report problems to Zabbix administrators
默认接收人：{TRIGGER.STATUS}:
默认信息：({HOST.NAME1}:{ITEM.KEY1}): {ITEM.VALUE1}
恢复主题：{TRIGGER.STATUS}:
恢复信息：({HOST.NAME1}:{ITEM.KEY1}): {ITEM.VALUE1}

条件：A and B

维护状态为非维护状态(Maintenance status not in maintenance)
触发器值=问题(Tigger value = PROBLEM)

操作：发消息给用户，可以指定某一时间段内发送给某一用户组或用，发送间隔等。

2. 发送邮件示例

创建步骤同上，配置–>动作–>事件源（触发器）–>创建动作。

不同的是，因为发送邮件没有字符限制，接受人、接受信息等可用默认，也可自定义。

二、创建报警媒介给用户发送报警

创建完触发的动作后，要想给用户发送告警，还需要创建报警媒介。

步骤：管理(Administration)–>报警媒介类型(Media types)

自带模板比较鸡肋，可以自己创建媒介，点击右上角创建

1. 创建媒介

名称：Send_Mail
类型：脚本（Script）
脚本名称：Send_Mail.py（此报警脚本需在Zabbix server自己编写，见下文）
脚本参数：需要传三个参数，目标邮件地址，邮件主题、邮件内容

2. 编辑报警脚本

编辑配置文件

[root@test ~]# vim /etc/zabbix/zabbix_server.conf
# 报警脚本路径
AlertScriptsPath=/etc/zabbix/alertscripts

创建脚本路径

[root@test ~]# mkdir -p /etc/zabbix/alertscripts/

克隆脚本代码

[root@test ~]# git clone https://github.com/fed0ra/template4zabbix.git
[root@test ~]# cp template4zabbix/zabbix-server/alertscripts/Send_Mail.py /etc/zabbix/alertscripts/
[root@test ~]# chmod +x /etc/zabbix/alertscripts/Send_Mail.py

3. 给用户指定报警媒介

步骤：管理–>用户–>报警媒介，选择创建好的的Send_Mail。

重启docker服务报错'device or resource busy'


Welcome to Fedor’s blog! If you find any problems, you can tell me in any way.

如果有容器在运行的时候重启docker服务，可能会导致容器无法启动， 错误信息类似于

Error response from daemon: Cannot start container zookeeper: Error getting container cad1fg3721g43gd128733827d3f4f8wwi82f1jusi8cf1d97fdc701076d2229j2 from driver devicemapper: Error mounting '/dev/mapper/docker-8:1-525372-cad1fg3721g43gd128733827d3f4f8wwi82f1jusi8cf1d97fdc701076d2229j2' on '/var/lib/docker/devicemapper/mnt/cad1fg3721g43gd128733827d3f4f8wwi82f1jusi8cf1d97fdc701076d2229j2': device or resource busy  
2016/05/25 04:42:07 Error: failed to start one or more containers

或者

d2859bd1f84b: Error pulling image (latest) from xxxxxx, Driver devicemapper failed to create image rootfs e6158e7962db43274de40fc3db65ad64811d43fe342dea633df20639f5a4e3cd: device e6158e7962db43274de40fc3db65ad64811d43fe342dea633df20639f5a4e3cd already exists 43fe342dea633df20639f5a4e3cd already exists c049b2b: Download complete  
e6158e7962db: Error downloading dependent layers

解决方法：找出没有umount的路径，命令如下：

cat /proc/mounts | grep "mapper/docker" | awk '{print $2}'

然后依次unmount
