日志收集分析系统（题外话）

日志的分析和监控在系统开发中占非常重要的地位，系统越复杂，日志的分析和监控就越重要，常见的需求有:

  • 根据关键字查询日志详情
  • 监控系统的运行状况
  • 统计分析，比如接口的调用次数、执行时间、成功率等
  • 异常数据自动触发消息通知
  • 基于日志的数据挖掘

  • 很多团队在日志方面可能遇到的一些问题有:

  • 开发人员不能登录线上服务器查看详细日志，经过运维周转费时费力

  • 日志数据分散在多个系统，难以查找
  • 日志数据量大，查询速度慢
  • 一个调用会涉及多个系统，难以在这些系统的日志中快速定位数据
  • 数据不够实时

  • 常见的一些重量级的开源Trace系统有:

  • facebook scribe

  • cloudera flume
  • twitter zipkin
  • storm
  • 这些项目功能强大，但对于很多团队来说过于复杂，配置和部署比较麻烦，在系统规模大到一定程度前推荐轻量级下载即用的方案；
    比如logstash+elasticsearch+kibana(LEK)组合；
    对于日志来说，最常见的需求就是收集、查询、显示，正对应logstash、elasticsearch、kibana的功能。

LEK介绍

logstash

  • logstash部署简单，下载一个jar就可以用了，对日志的处理逻辑也很简单，就是一个pipeline的过程：

    inputs >> codecs >> filters >> outputs.

  • logstash支持常见的日志类型，与其他监控系统的整合也很方便，可以将数据输出到zabbix、nagios、email等。

  • 推荐用redis作为输入缓冲队列。
  • 你还可以把数据统计后输出到graphite，实现统计数据的可视化显示。(metrics demo,statsd,graphite)

  • elasticsearch

elasticsearch是基于lucene的开源搜索引擎，近年来发展比较快，主要的特点有:

  • real time
  • distributed
  • high availability
  • document oriented
  • schema free
  • restful api

  • kibana

kibana是一个功能强大的elasticsearch数据显示客户端，logstash已经内置了kibana；
你也可以单独部署kibana，最新版的kibana3是纯html+js客户端，可以很方便的部署到Apache、Nginx等Http服务器。

安装部署

1、安装java(推荐1.7+版本)

java -version

安装略

2、安装redis

wget http://download.redis.io/releases/redis-3.0.6.tar.gz
tar xf redis-3.0.6.tar.gz
cd redis-3.0.6
make
make install

可以通过redis源代码里utils/install_server下的脚本简化配置工作

cd utils
sudo ./install_server.sh

install_server.sh在问你几个问题后会把redis安装为开机启动的服务，可以通过下面的命令行来启动/停止服务

/etc/init.d/redis_6379 start/stop

启动redis客户端来验证安装

redis-cli
redis> set foo bar
OK
redis> get foo
"bar

3、安装Elasticsearch 2.1.1

下载解压、建立软连接：

cd /usr/local/src/
wget https://download.elasticsearch.org/elasticsearch/release/org/elasticsearch/distribution/tar/elasticsearch/2.1.1/elasticsearch-2.1.1.tar.gz
tar xf elasticsearch-2.1.1.tar.gz
mv elasticsearch-2.1.1 /usr/local/
cd /usr/local/
ln -s /usr/local/elasticsearch-2.1.1/ /usr/local/elasticsearch

修改配置文件：

cd elasticsearch/config
vim elasticsearch.yml

配置文件内容如下：

cluster.name: my-application
node.name: node-1
bootstrap.mlockall: true
network.host: 192.168.2.42
http.port: 9200

启动elasticsearch

cd elasticsearch-2.1.1/bin/
nohup ./elasticsearch &

or

/usr/local/elasticsearch/bin/elasticsearch -d

查看elasticsearch启动是否成功

curl -X GET http://192.168.2.42:9200
netstat -lntup | grep 9200

4、安装Logstash 2.1.1

下载Logstash 2.1.1，建立软连接

cd /usr/local/src/
wget https://download.elastic.co/logstash/logstash/logstash-2.1.1.tar.gz
tar xf logstash-2.1.1.tar.gz
mv logstash-2.1.1 /usr/local/
cd /usr/local/
ln -s /usr/local/logstash-2.1.1 /usr/local/logstash

通过service启动logstash，启动文件项目地址https://github.com/unixhot/elk-config

git clone https://github.com/unixhot/elk-config.git
cd elk-config
cp logstash.init /etc/init.d/logstash
chmod +x /etc/init.d/logstash

编辑配置文件，根据环境修改

vim /etc/logstash.conf
/usr/local/logstash/bin/logstash -f /etc/logstash.conf --configtest
/etc/init.d/logstash start

测试logstash

/opt/logstash/bin/logstash -e 'input { stdin{} } output { stdout{ codec => rubydebug} }'
hello world

logstash把数据写入es中

/opt/logstash/bin/logstash -e 'input { stdin{} } output {  elasticsearch { hosts => "your ip address"  }  stdout{} }'

通过浏览器访问elasticsearch，查看logstash key下是否有新插入的数据：IP:9200/_plugin/head

通过-f参数指定配置文件启动

/opt/logstash/bin/logstash -f /etc/logstash.conf

编写配置文件配置文件（例）

cat /etc/logstash.conf

#输入
input {
    file {
        path => "/path/to/your/logs/*"
        type => "type_1"
        #start_position => "beginning"
        #multiline多行，用于java日志
        codec => multiline {
            pattern => "^\["
            negate => true
            what => previous
        }
    }
    file { 
        path => "/path/to/your/logs2/*"
        type => "type_2"
        codec => multiline {
            pattern => "^%{TIMESTAMP_ISO8601}"
            negate => true
            what => previous
        }
    }

}

#输出
output {
    #根据key类型判断，写入到elasticsearch的哪一个key中
    if [type] == "type_1"  {
        elasticsearch {
            hosts => "192.168.2.42"
            index => "type_1-%{+YYYY-MM}"
        }
    }
    if [type] == "type_2"  {
        elasticsearch {
            hosts => "192.168.2.42"
            index => "type_2-%{+YYYY-MM}"
        } 
    }

}

测试：匹配分、时、[

input {
    stdin {
        type => "test01"
        #start_position => "beginning"
        codec => multiline {
            pattern => "^[0-1][0-9]:[0-5][0-9]|2[0-3]:[0-5][0-9]|^\["
            negate => true
            what => previous
        }
    }
 }
output {
    if [type] == "test01"  {
        elasticsearch {
            hosts => "192.168.2.42"
             index => "test01-%{+YYYY-MM}"
         }
    }
    stdout {
        codec => rubydebug
    }
}

5、安装Kibana 4.3.1(Compatible with Elasticsearch 2.1.x)

下载安装kibana4.3.1

cd /usr/local/src/
wget https://download.elastic.co/kibana/kibana/kibana-4.3.1-linux-x64.tar.gz
tar xf kibana-4.3.1-linux-x64.tar.gz
mv kibana-4.3.1-linux-x64/ /usr/local/
ln -s /usr/local/kibana-4.3.1-linux-x64/ /usr/local/kibana

修改启动文件中elasticsearch的url地址

vim /usr/local/kibana/config/kibana.yml

elasticsearch.url: “http://192.168.2.42:9200“

启动kibana

screen kibana
cd kibana-4.3.1-linux-x64/bin
./kibana
#nohup ./kibana &

查看启动是否成功

netstat -lntup | grep 5601

访问kibana

http://192.168.2.42:5601/

nginx代理kibana配置验证
生成用户认证的用户名和密码

htpasswd -c /usr/local/nginx/conf/kibana.auth admin
ll /usr/local/nginx/conf/kibana.auth

为Nginx添加auth认证配置

编辑nginx配置文件：

vim /etc/nginx/nginx.conf

添加如下：

include extra/kibana.conf;

编辑nginx配置文件：

vim /etc/nginx/extra/kibana.conf

添加如下：

server {
    listen    80;
    server_name    kibana.example.com;
    access_log    /var/log/nginx/kibana.srv-log-dev.log;
    error_log    /var/log/nginx/kibana.srv-log-dev.error.log;

    #ssl    on;
    #ssl_certificate    /etc/nginx/ssl/all.crt;
    #ssl_certificate_key    /etc/nginx/ssl/server_nopass.key;

    location / {
    #location ~ ^/kibana4/.* {
        proxy_pass    http://kibana4;
        rewrite ^/kibana4/(.*) /$1 break;
        auth_basic "Authorized users only,for search...";
        auth_basic_user_file    /usr/local/nginx/conf/kibana.auth;
        proxy_set_header    X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header    Host $host;
        #auth_basic    "Restricted";
        #auth_basic_user_file    /etc/nginx/conf.d/kibana.myhost.org.htpasswd;
    }
}

重启nginx

/usr/sbin/nginx -t
/usr/sbin/nginx -s reload

访问nginx

http://kibana.example.com

Windows下使用 GitHub Pages + Hexo搭建博客

Welcome to fed0ra’s blog! If you found any problems, you can tall me on GitHub.

版权声明

严重参考（照抄）http://www.jianshu.com/p/985d07d88ef4#
http://www.chinaz.com/web/2016/0105/491998.shtml

为什么选择GitHub Pages

  • 1 GitHub Pages 有300M免费空间，资料自己管理，保存可靠；
  • 2 学着用 GitHub ，享受 GitHub 的便利，上面有很多大牛，眼界会开阔很多；
  • 3 顺便看看 GitHub 工作原理，最好的团队协作流程；
  • 4 GitHub 是趋势；
  • 5 你不觉得一个文科生用 GitHub 很geek吗？瞬间跻身技术界；
  • 6 就算 GitHub 被墙了，我可以搬到国内的 GitCafe 中去。

一、首先你需要准备什么

  • 一个github账户

访问：https://www.github.com
注册十分简单，记住注册时使用的邮箱，过程略。

  • 创建一个仓库

注册号账户后，在你的主页面点击New repository

Repository name仓库名字命名为yourname.github.io（yourname自定义，后面.github.io最后不变，我这里为fed0ra.github.io）

免费只能勾选public，下面勾选Initialize this repository with a README

点击Create repository就OK了

现在你访问你的yourname.github.io这个域名是404是正确的。

二、环境安装

  • Node.js（下载Windows版本后安装，安装过程一路向西）
  • git（下载Windows版本后安装，安装过程同上）（不翻墙下载速度比较慢，也可以去百度下载git）

三、使用Hexo生成博客的静态页面

Hexo 是一个轻量的静态博客框架。通过Hexo可以快速生成一个静态博客框架,仅需要几条命令就可以完成,相当方便。

1、运行Node.js command prompt

安装好Node.js后，在桌面或开始菜单中找到Node.js command prompt的快捷方式，打开后输入命令

$ cd d:
$ npm install hexo-cli -g

cd d:是切换到nodejs安装路径，不必要操作看个人习惯

2、Hexo部署

通过命令hexo init [文件名]生成

$ f:
$ hexo init fed0ra.github.io

cd f:切换到任意盘符下，这里切换到f盘下；fed0ra.github.io根据自己网站定义(yourname.github.com)，记下之后再github建库时候建成同名的库；回车后仔细看有没有报错，没有报错证明成功

这是你会在F盘根目录下，发现多出一个名字为fed0ra.github.io的文件夹，再输入以下命令安装依赖文件

$ cd fed0ra.github.io
$ npm install

回车后看是否报错

经过以上方法后，Hexo会自动在目标文件夹建立博客网站所需要的所有文件。

此时可以通过输入以下命令在本地进行预览（在刚才创建的文件夹里）：

$ hexo generate (可以简写成: hexo g生成文件，出现一排排INFO表示成功)
$ hexo server （启动服务，成功后显示一行INFO，本地浏览地址）

此时打开浏览器，在浏览器地址栏输入 http://localhost:4000/ （默认端口为4000）便可以看到最原始的博客了（以后发表博文想先预览，也可以通过 hexo server 在本地先跑起来，看看效果，Ctrl+C停止服务，停止后本地就访问不了了）

到目前为止博客的雏形已经有了。

在Node.js command prompt中按 Ctrl + C并输入y可以停止该服务（本地访问不了了）。

四、将本地文件部署到 GitHub

1、配置和使用 Github

使用ssh keys让本地git项目与远程的github建立联系呢（等同于把上一步生成的页面文件，即F盘下的fed0ra.github.io文件夹的内容上传的github库）

第二部安装好git后会生成快捷方式Git Bash，运行Git Bash，在命令行操作

a）输入以下命令检查本机是否有ssh密钥：

$ cd ~/.ssh

如果提示：No such file or directory 说明你是第一次使用git,还没有生成秘钥

b）输入以下命令生成新的SSH Key：

$ ssh-keygen -t rsa -C "邮件地址@youremail.com"

一直回车

注意: 此处的邮箱地址，你可以输入自己的邮箱地址;此处的「-C」的是大写的「C」

c）查看生成的秘钥

$ cd ~/.ssh
$ ls
$ cat id_rsa.pub

cd ~/.ssh进入生成秘钥的目录，ls查看，发现目录下有id_rsa和id_rsa.pub,id_rsa.pub就是我们需要的，cat id_rsa.pub查看文件里的内容，选中后已经复制好了，直接粘贴到其他文档中备用。

d）添加SSH Key到GitHub

在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。

登陆github系统。点击右上角的Account Settings—>SSH keys —>new SSH key。

把你本地生成的密钥刚刚复制好的粘贴到里面(key文本框中)，点击add key就ok了。

测试：

可以输入下面的命令，看看设置是否成功，git@github.com的部分不要修改：

$ ssh -T git@github.com

如果是下面的反馈：

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.

RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.

Are you sure you want to continue connecting (yes/no)?

不要紧张，输入yes就好，然后会看到：

Hi aierui! You’ve successfully authenticated, but GitHub does not provide shell access.

e）最后一击，设置用户信息

现在你已经可以通过 SSH 链接到 GitHub 了，还有一些个人信息需要完善，否则后面本地文件上传github会出错。

$ git config --global user.name "yourname"//用户名
$ git config --global user.email "imland@outlook.com"//填写自己的邮箱

2、修改 Hexo 中的 _config.yml 文件

在fed0ra.github.com文件夹下找到_config.yml文件

找到其中的 deploy 标签，改成如下所示，并保存：

deploy:
  type: git
  #repository: https://github.com/fed0ra/fed0ra.github.io
  repository: ssh://git@github.com/fed0ra/fed0ra.github.io
  branch: master
  plugins: -hexo-generator-feed

其中 branch 选项可以填写 master 或 gh-pages。如果你是为一个项目制作网站，那么需要把 branch 设置为 gh-pages 。若要绑定自定义域名也可以参考Hexo 或 Github Page 的帮助文档，制作一个 CNAME 文件。

3、将第三步Hexo生成博客的静态页面部署到github仓库

运行Git Bash，输入以下命令：

$ cd f:    \\切换到F盘
$ cd fed0ra.github.io    \\切换到博客目录
$ hexo clean
$ hexo generate
$ hexo deploy

如果出现ERROR Deployer not found: github，将deploy的type改成 git，然后再在Git Shell中运行以下命令：

$ npm install hexo-deployer-git --save

再重新来一遍

$ hexo clean
$ hexo generate
$ hexo deploy

运行没报错，出现INFO Deploy done: git则表示博客已经部署到github上了。

去https://github.com/fed0ra/fed0ra.github.io仓库查看，文件已经上传上了。再访问http://fed0ra.github.io，显示初始博客页面，大功告成。
